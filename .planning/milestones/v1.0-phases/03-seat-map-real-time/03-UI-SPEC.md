## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage in Phase 3 |
|-------|-------|-------------------|
| xs | 4px | Icon-text gap in legend chips, seat label padding, tooltip internal padding |
| sm | 8px | Legend chip internal spacing, zoom control button gaps, bottom sheet seat row gaps |
| md | 16px | Seat panel internal padding, calendar-to-showtime gap, form field gaps |
| lg | 24px | Section gaps (date picker to seat map), legend bar horizontal padding, side panel section separators |
| xl | 32px | Layout gap between seat map area and side panel, page-level vertical padding |
| 2xl | 48px | Gap between booking header and content area |
| 3xl | 64px | Not used in Phase 3 (booking page is dense) |

Exceptions:
- Seat SVG circle/rect: sizes defined by source SVG, not spacing tokens
- Touch target on mobile seat tap: minimum 44px per WCAG (achieved via SVG viewBox scaling, not CSS)
- Countdown timer bar height: 40px (fixed, sticky at top of content)
- Bottom sheet handle area: 20px (drag indicator zone)
- Zoom control button: 40px x 40px each (+ / - / reset)
- Side panel width (desktop): 360px fixed
- Bottom sheet collapsed height: 72px (shows summary: "{n}석 선택 | {total}원")
- Bottom sheet expanded max-height: 60vh

**Source:** globals.css @theme --spacing-* tokens (Phase 1)

|------|-------|-------|

| Secondary (30%) | #F5F5F7 (Gray-100) | Seat map container background (provides contrast for SVG), date picker area background, legend bar background |

### Accent (#6C3CE0) reserved for:
1. "다음" CTA button (primary action to proceed to payment)
2. Selected showtime chip active state (bg: primary, text: white)
3. Calendar selected date indicator (bg: primary, text: white)
4. Side panel total price highlight border-left (3px solid)
5. Countdown timer bar background (normal state, above 3 minutes)

### Seat Tier Color Palette (D-07)

These are seat map domain colors, separate from the brand palette. Admin-configurable per performance, with these defaults:

| Tier | HEX | RGB | Usage |
|------|------|-----|-------|
| VIP | #6C3CE0 | rgb(108, 60, 224) | VIP tier seats (matches brand primary) |
| R | #3B82F6 | rgb(59, 130, 246) | R tier seats (blue) |
| S | #22C55E | rgb(34, 197, 94) | S tier seats (green) |
| A | #F59E0B | rgb(245, 158, 11) | A tier seats (amber) |

### Seat State Colors (D-08, D-09)

| State | Fill | Stroke | Stroke Width | Additional |
|-------|------|--------|-------------|------------|
| Available | Tier color (see above) | none | 0 | cursor: pointer, opacity: 1 |
| My selection | Tier color | #1A1A2E (Gray-900) | 3px | Checkmark overlay icon (white, centered, 12px) |
| Other user locked | #D1D5DB (Gray-300) | none | 0 | cursor: not-allowed, opacity: 0.6 |
| Sold (permanent) | #D1D5DB (Gray-300) | none | 0 | cursor: not-allowed, opacity: 0.6 |
| Hover (available) | Tier color | Tier color | 2px | brightness(1.15) filter, cursor: pointer |

## Interaction Contracts

### Seat Map Interactions (SEAT-01 through SEAT-06)

| Interaction | Behavior |
|-------------|----------|
| Seat tap/click (available) | Immediately changes to "my selection" state (tier color + dark stroke + checkmark). API call POST /seats/lock fires in parallel. If lock fails (409), revert visual state + show toast "이미 다른 사용자가 선택한 좌석입니다". Optimistic UI. |
| Seat tap/click (my selection) | Immediately reverts to "available" state. API call DELETE /seats/lock/:lockId fires. Side panel updates. |
| Seat tap/click (locked/sold) | No visual change. No API call. cursor: not-allowed prevents interaction feedback. |
| Seat hover (desktop, available) | Tooltip appears showing "{등급명} {열}{번} - {가격}원". Seat applies brightness(1.15) filter + 2px stroke in tier color. |
| Max seats reached (4) + tap new seat | Toast "최대 4석까지 선택할 수 있습니다. 다른 좌석을 먼저 해제해주세요." No seat state change. |

### Zoom/Pan Controls (SEAT-02, SEAT-03)

| Interaction | Behavior |
|-------------|----------|
| Pinch zoom (mobile) | react-zoom-pan-pinch handles natively. Min scale: 0.5x. Max scale: 4x. Smooth animation |
| Wheel zoom (desktop) | Zooms centered on cursor position. Same min/max scale. |
| Pan/drag | Available at any zoom level. Bounded to SVG content area (no over-scrolling into empty space). |
| Zoom in button (+) | Increments scale by 0.5x per click. Disabled at max scale (4x). |
| Zoom out button (-) | Decrements scale by 0.5x per click. Disabled at min scale (0.5x). |
| Reset button (Maximize2 icon) | Animates to fit-all view (scale 1x, centered). 200ms ease-out transition. |
| Initial load | Seat map renders at scale-to-fit (entire map visible within container). No animation needed on first render. |

### Date/Showtime Selection (BOOK-01, BOOK-02)

| Interaction | Behavior |
|-------------|----------|
| Calendar date tap | Highlights date in primary color. Fetches showtimes for that date. Previous showtime selection clears. |
| Calendar disabled date | Grayed out, not tappable. No showtimes exist for this date. |
| Calendar month navigation | Prev/Next chevron buttons. Only months with any showtime dates are navigable (no month without data). |
| Showtime chip tap | Highlights chip (primary bg + white text). Fetches seat availability via API + joins WebSocket room for this showtime. Previous seat selections cleared + locks released. |
| No showtimes for date | Shows "선택한 날짜에 예정된 회차가 없습니다" in place of chip row. |

### Real-Time Updates (SEAT-06, D-12, D-13)

| Interaction | Behavior |
|-------------|----------|
| Other user locks seat | Seat instantly changes to gray (locked state). No animation per D-12. If the user was hovering that seat, tooltip dismisses. |
| Other user unlocks seat | Seat instantly changes to tier color (available state). No animation per D-12. |
| Race condition (D-13) | User taps available seat -> optimistic UI -> server returns 409 -> seat reverts to gray + toast "이미 다른 사용자가 선택한 좌석입니다" |
| WebSocket disconnect | Toast "실시간 연결이 끊어졌습니다. 재연결 중..." persists until reconnected. Socket.IO handles exponential backoff reconnection automatically. |
| WebSocket reconnect | Dismiss disconnect toast. Show success toast "실시간 연결이 복구되었습니다" for 3 seconds. Refetch full seat state to catch missed events. |
| Showtime switch | Leave previous WebSocket room. Join new room. Clear all local seat state. Fetch fresh seat snapshot. |

### Countdown Timer (D-10, D-11)

| Interaction | Behavior |
|-------------|----------|
| Timer start | Begins when first seat is successfully locked (server returns lock expiry timestamp). Displays in BookingHeader. |
| Timer tick | Updates every 1 second. Format: MM:SS. Uses monospace font for stable width. |
| Timer > 3 min | Primary color background (#6C3CE0), white text, Clock icon. |
| Timer <= 3 min | Transitions to error color background (#C62828), white text, AlertTriangle icon. Transition: 300ms background-color ease. |
| Timer expires (0:00) | Opens TimerExpiredModal (AlertDialog, non-dismissible). All seat locks have expired server-side. |
| Timer expired CTA "처음으로" | Clears all local state (selected seats, showtime, date). Resets page to initial state (date picker with no selection). Closes modal. |
| Additional seat lock | Timer does NOT reset. All seats share the same expiry (first lock's TTL). Timer shows the earliest expiry. |

### Side Panel / Bottom Sheet (D-04, D-05)

| Interaction | Behavior |
|-------------|----------|
| Seat selected | Seat row appears in panel list with slide-down animation (150ms ease-out). Total updates instantly. |
| Seat removed (X button in panel) | Seat row removed with fade-out (100ms). Corresponding seat on map reverts to available. Lock released via API. |
| "다음" button tap | Validates: at least 1 seat selected. Navigates to Phase 4 payment entry (route TBD, placeholder console.log in Phase 3). |
| "다음" button disabled state | Text: "좌석을 선택해주세요". Gray background (#D1D5DB), gray text (#71717A). Disabled when selectedSeats.length === 0. |
| Bottom sheet drag (mobile) | Drag handle at top (centered gray bar, 40px wide, 4px tall, rounded-full). Drag up expands to max 60vh. Drag down collapses to 72px. Snap points: collapsed (72px) and expanded (60vh). |
| Bottom sheet auto-expand | On first seat selection, bottom sheet auto-expands to show the seat. After that, user controls expansion. |

### Booking Page Navigation

| Interaction | Behavior |
|-------------|----------|
| Back button (BookingHeader) | Navigates to performance detail page. No confirmation dialog. Seat locks auto-release via TTL. |
| Browser back | Same as back button. No beforeunload warning in Phase 3 (deferred to Phase 5). |
| Performance title click | No action (not a link on booking page). Display only. |

## Discretion Decisions

Areas marked as "Claude's Discretion" in Phase 3 CONTEXT.md, resolved here:

| Area | Decision | Rationale |
|------|----------|-----------|
| react-zoom-pan-pinch config | Min scale 0.5x, max 4x, smooth animation enabled. Initial scale: fit-to-container (centerOnInit: true). Double-tap zoom: disabled (conflicts with seat tap selection) | Standard range for seat map viewing. Double-tap disabled to prevent accidental zoom when tapping seats quickly |
| WebSocket namespace/room | Namespace: `/booking`. Room: `showtime:{showtimeId}`. Join room on showtime selection, leave on showtime change or page exit | Per-showtime rooms minimize broadcast scope. Only users viewing the same showtime receive updates |
| Redis key structure | `seat:{showtimeId}:{seatId}` with value `{userId}`, TTL 600s. Example: `seat:abc123:A-1 = user456` | Matches architecture doc pattern. Simple, scannable, debuggable |
| SVG rendering optimization | Fetch SVG text from R2 URL, parse with DOMParser, convert seat elements to React components. Non-seat elements (stage, labels, decorations) rendered as static SVG. Only seat elements are interactive React components. Memoize non-seat SVG portion | Avoids re-rendering 1000+ elements on every state change. Only changed seats re-render via React reconciliation |
| Timer start trigger | Timer starts on first successful seat lock (not on page entry). If user deselects all seats, timer continues (locks may still be releasing). Timer only resets on expiry or page reset | Starting on page entry would waste lock time while user browses dates. Starting on first lock aligns timer with actual server TTL |
| Bottom sheet implementation | Custom implementation using CSS transforms + touch event handling. No external library. Snap points: collapsed (72px) and expanded (60vh). Drag velocity detection for snap direction | Bottom sheet libraries add bundle weight for simple snap behavior. CSS transform + translateY with touch events is lightweight and sufficient. shadcn Sheet is side-drawer only, not suitable for bottom snap behavior |
| "다음" button connection to Phase 4 | In Phase 3, the "다음" button logs the booking payload to console (`{ showtimeId, selectedSeats, totalPrice }`) and shows a toast "결제 기능은 준비 중입니다". Phase 4 will replace with actual navigation to payment page | Clean boundary. Phase 3 proves seat selection works end-to-end. Phase 4 adds payment |
