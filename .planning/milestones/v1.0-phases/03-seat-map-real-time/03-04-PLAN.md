**Gap 2 — Locked seat click toast (Test 11, major):**

In `seat-map-viewer.tsx`, modify `handleClick` (line 106-124):
- Current condition (line 119): `if (state === 'available' || isSelected)` silently ignores locked/sold seats.

```
const state = seatStates.get(seatId) ?? 'available';
```

```typescript
    style: { backgroundColor: '#F3EFFF', color: '#6C3CE0' },
  });
  return;
}
```
- Add `seatStatesMap` to the useCallback dependency array.
- This allows the locked-seat toast to fire. The `!selectedSeatIds.has(seatId)` guard ensures users can still deselect their own locked seats.

**Gap 3 — Mobile layout (Test 12, major). Four sub-fixes:**

**(A) booking-page.tsx — flex direction + bottom padding:**

Line 357, change the inner flex container:
- FROM: `<div className="flex gap-8">`
- TO: `<div className="flex flex-col lg:flex-row lg:gap-8">`

This makes the left column (seat map area) stack vertically on mobile and go side-by-side on desktop.

Also on the `<main>` tag (line 356), add bottom padding for the bottom sheet:
- FROM: `className="mx-auto w-full max-w-[1280px] px-4 py-4 lg:px-6 lg:py-8"`
- TO: `className="mx-auto w-full max-w-[1280px] px-4 py-4 pb-24 lg:px-6 lg:py-8 lg:pb-8"`

The `pb-24` (96px) ensures content is never hidden behind the 72px collapsed bottom sheet + some breathing room. On desktop (lg:pb-8), the bottom sheet is hidden so normal padding applies.

Also in the loading skeleton (around line 318), apply the same flex-col fix:
- FROM: `<div className="flex gap-8">`
- TO: `<div className="flex flex-col lg:flex-row lg:gap-8">`

**(B) date-picker.tsx — responsive day cell size:**

In the classNames object:
- `day`: change `size-10` to `size-8 sm:size-10` — 32px cells on small screens, 40px on sm+.
- `day_button`: change `size-10` to `size-8 sm:size-10` — match the day cell.
- `weekday`: keep as-is (text only, no size issue).

This prevents the 7-column calendar from overflowing a 320px viewport (7 x 40px = 280px + padding would exceed; 7 x 32px = 224px fits comfortably).

**(C) seat-map-controls.tsx — z-index above bottom sheet:**

Change z-index class:
- FROM: `className="absolute bottom-4 right-4 z-10 flex flex-col gap-2"`
- TO: `className="absolute bottom-4 right-4 z-50 flex flex-col gap-2"`

The bottom sheet uses z-40. Zoom controls must be above it (z-50) so they remain clickable on mobile. On desktop the sheet is lg:hidden so this has no visual impact.

**(D) seat-selection-sheet.tsx — safe area handling:**

On the outermost div (line 107), add safe area bottom padding for iOS notch devices:

- TO: `className="fixed bottom-0 left-0 right-0 z-40 rounded-t-2xl border-t bg-white pb-safe shadow-[0_-4px_20px_rgba(0,0,0,0.1)] lg:hidden"`

Note: `pb-safe` may not be a default Tailwind utility. Check if it exists in the project's Tailwind config. If not, use inline style `paddingBottom: 'env(safe-area-inset-bottom)'` on the div instead.
