## Spacing Scale

Declared values (multiples of 4). Already established in `globals.css` @theme block:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, mobile gutter |
| lg | 24px | Section padding, desktop gutter |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions for Phase 5:
- **44px minimum touch target**: All interactive elements (buttons, links, inputs, tab bar items) on mobile must have a minimum tappable area of 44x44px. Use `min-h-[44px] min-w-[44px]` or equivalent padding to achieve this. This is a WCAG 2.5.5 requirement (D-04).
- **Bottom tab bar height**: 56px total (44px touch area + 12px safe-area padding via `pb-safe`). Use `env(safe-area-inset-bottom)` for iOS notch devices.
- **Sticky footer CTA**: 64px height on mobile booking flow pages where applicable.

## Component Inventory — Phase 5 New Components

### 1. MobileTabBar

Bottom navigation bar replacing GNB on mobile (< md breakpoint, 768px).

**Specifications:**
- Height: 56px + `env(safe-area-inset-bottom)` for iOS safe area
- Background: #FFFFFF with top border 1px `--color-border` (#E4E4E7)
- Position: fixed bottom, z-50, full width
- 4 tabs: equal width flex distribution
- Each tab: icon (20px) above label (10px), centered vertically
- Active state: icon and label in `--color-primary` (#6C3CE0), weight 600
- Inactive state: icon in `--color-gray-400` (#A1A1AA), label in `--color-gray-500` (#71717A), weight 400
- Touch target: 44px minimum per tab (full tab area is tappable)
- No animation on tab switch (instant color change)
- Visible on all public pages. Hidden on: /admin/*, /booking/*/confirm, /booking/*/[seat selection]
- When MobileTabBar is visible, add `pb-[56px]` (or pb-[68px] with safe area) to body/main content to prevent overlap

**Interaction with existing GNB:**
- Desktop (>= md): Show existing GNB header (sticky top). Hide MobileTabBar.
- Mobile (< md): Hide GNB header. Show MobileTabBar (fixed bottom). Hide existing mobile hamburger menu.

Source: CONTEXT.md D-03, docs/04-UIUX-GUIDE.md Mobile Nav pattern

### 2. NetworkBanner

Full-width banner for offline/timeout states.

**Specifications:**
- Height: 44px
- Background: `--color-error` (#C62828)
- Text: white, 14px, weight 600, centered
- Position: fixed top (above GNB on desktop, above content on mobile), z-[60]
- Contains: message text + "다시 시도" button (white outline, 14px)
- Button: border 1px white, background transparent, text white, rounded-md, px-3 h-8
- Appears on: `navigator.onLine === false` or fetch timeout
- Disappears on: network restoration (online event) or successful retry
- Push content down when visible (no overlap)

Source: CONTEXT.md D-11

### 3. NotFoundPage

Custom 404 page at `apps/web/app/not-found.tsx`.

**Specifications:**
- Layout: vertically and horizontally centered in viewport
- Visual: "( ._.)" text face, 48px, `--color-gray-400`, mb-6
- Heading: "페이지를 찾을 수 없습니다", 20px weight 600, `--color-gray-900`, mb-2
- Body: "요청하신 페이지가 존재하지 않거나 이동되었습니다.", 16px weight 400, `--color-gray-500`, mb-6
- CTA: Button variant="default" linking to "/", text "홈으로 돌아가기"

## Mobile Responsive Patterns

All public pages must render correctly below md breakpoint (768px). Admin pages remain desktop-only (D-01).

### Breakpoints (existing Tailwind v4 defaults)

| Name | Min Width | Usage |
|------|-----------|-------|
| sm | 640px | Minor mobile adjustments |
| md | 768px | GNB/TabBar switch, layout shifts |
| lg | 1024px | Full desktop layout |
| xl | 1280px | Max-width container (1200px) |

### Page-Specific Mobile Adaptations

**Home Page (D-01):**
- Banner carousel: full width, aspect-[2.5/1] on mobile (same as desktop)
- Genre grid: 2x4 grid on mobile (4 columns), scrollable row on desktop
- Hot/New sections: horizontal scroll with snap on mobile, grid on desktop
- Section padding: px-4 on mobile, px-6 on desktop

**Genre Page (D-01):**
- Performance cards: 2-column grid on mobile (gap-3), 4-column on desktop (gap-4)
- Sub-category chips: horizontal scroll on mobile
- Sort toggle: full width on mobile

**Performance Detail (D-05):**
- Desktop: 2-column layout (poster left, info right)
- Mobile: single column. Poster full-width (aspect-[2/3], max-h-[400px] object-cover). Info below poster.
- Tabs: full width, horizontal scroll if overflow on mobile
- Tab content: px-4 on mobile, px-0 on desktop (within container)

**Search Page (D-01):**
- Search input: full width on mobile with px-4
- Results: single column card list on mobile, 2-column grid on desktop
- Filter chips: horizontal scroll on mobile

**Booking Seat Selection (D-06):**
- Date/showtime selector: collapsible accordion at top on mobile. Tap to expand calendar. Collapsed by default to maximize seat map space.
- Seat map: full viewport width on mobile (no side padding)
- Seat selection panel: bottom sheet (existing seat-selection-sheet.tsx) on mobile, side panel on desktop
- Seat map controls (+/-/reset): floating bottom-right on mobile, 44px touch targets

**Booking Confirm (D-02):**
- Order summary: single column stack on mobile
- Payment widget: full width on mobile

**Booking Complete:**
- Centered layout, same on mobile and desktop (already responsive)

**My Page / Reservations (D-02):**
- Desktop: table layout for reservation list
- Mobile: card-based list. Each reservation card shows: performance name, date, seat count, status badge, price. Tap to navigate to detail.
- Reservation detail: single column stack on mobile

**Auth Pages:**
- Already responsive (form-based, single column)

## Error Handling Architecture

### Error Interceptor (D-10)

Enhance `apps/web/lib/api-client.ts` to categorize errors:

| HTTP Status | Category | UI Treatment |
|-------------|----------|-------------|
| 400 | Validation | Sonner error toast with server message |
| 401 | Auth expired | Redirect to /auth (existing) |
| 403 | Forbidden | Sonner error toast: "접근 권한이 없습니다." |
| 404 | Not found | Sonner error toast: "요청하신 정보를 찾을 수 없습니다." |
| 408/timeout | Timeout | Sonner error toast with timeout message |
| 429 | Rate limit | Sonner error toast: "요청이 너무 많습니다. 잠시 후 다시 시도해주세요." |
| 500+ | Server error | Sonner error toast with server error message |
| Network error | Offline | NetworkBanner component (not toast) |

Error toast duration: 5000ms for errors (override default 3000ms).

### Error Code System (D-12)

Format: `ERR-{HTTP_STATUS}` (e.g., ERR-400, ERR-500).

Display in toast as secondary text below main message:
```
일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요.
오류 코드: ERR-500
```

Error code is 12px, `--color-gray-500`, displayed below main error message in toast.

### Error Boundaries

| Boundary | Location | Behavior |
|----------|----------|----------|
| Global | `apps/web/app/error.tsx` | Existing. Enhance with error code display. |
| Not Found | `apps/web/app/not-found.tsx` | New. Custom 404 page (see component spec above). |
| Page-level | Each route `error.tsx` | Not required for Phase 5. Global boundary is sufficient. |

## Interaction Contracts

### Bottom Tab Bar Navigation
- Tap tab: instant navigation via Next.js router.push
- Active tab determined by `usePathname()` match
- Category tab: navigates to `/genre/musical` (default genre)
- Search tab: navigates to `/search`
- No swipe gestures between tabs

### Skeleton Loading Transitions
- Skeleton renders immediately on page load (no delay)
- Data replaces skeleton with no additional animation (React Query handles swap)
- Use React Suspense boundaries where applicable for streaming
- Skeleton pulse animation: existing shadcn `animate-pulse` (CSS only, no JS)

### Network Banner
- Appears within 1 second of offline detection (`navigator.onLine` + `online`/`offline` events)
- "다시 시도" button: calls `window.location.reload()`
- Banner disappears automatically when `online` event fires
- Banner pushes content down (not overlay)

### Error Toast Behavior
- Sonner toast position: top-center (existing)
- Error toasts: 5000ms duration
- Success toasts: 3000ms duration (existing)
- Maximum 3 toasts visible simultaneously (Sonner default)
- Toast includes error code as description (secondary line)
