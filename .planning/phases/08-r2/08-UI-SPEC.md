### Image Display Contract

All `<Image>` and `<img>` tags displaying poster/banner/SVG content MUST:

1. Accept both absolute R2 URLs (`https://...r2.dev/...`) and relative local paths (`/seed/...`) during transition
2. Use `next/image` `<Image>` component for poster and banner images in production (benefits from optimization)
3. Use native `<img>` or `fetch` for SVG seat maps (SVG must be loaded as DOM, not optimized by Next.js Image)
4. Handle image load errors gracefully — no broken image icons visible to user

**Image load failure fallback spec:**
- Icon: `ImageOff` from lucide, 24px size, color `text-gray-400`
- Layout: container uses `flex items-center justify-center` to center the icon both horizontally and vertically
- Background: `bg-gray-100` filling the full container dimensions
- Container dimensions: poster 160px x 240px, banner aspect-ratio preserved within form width, SVG full-width within preview container

## Interaction States

### Upload Flow (existing — preserved as-is)

```
[Idle] → Click drop zone / drag file
  ↓
[Validating] → Check file size (5MB poster / 10MB SVG / 5MB banner)
  ↓ fail → toast.error with size message, return to [Idle]
  ↓ pass
[Requesting Presigned URL] → POST /api/v1/admin/upload/presigned
  ↓ fail → toast.error "업로드에 실패했습니다.", return to [Idle]
  ↓ pass
[Uploading to R2] → PUT presigned URL with file body + Content-Type header
  ↓ fail → toast.error "업로드에 실패했습니다. 다시 시도해주세요.", revert preview, return to [Idle]
  ↓ pass
[Complete] → toast.success, show preview, set form value to publicUrl
```

### Image Loading (R2 URLs in production)

```
[Loading] → <Image> or <img> fetches from R2 public URL
  ↓ success → Display image normally
  ↓ fail (404 / CORS / network) → Show gray-100 container with ImageOff icon (24px, gray-400, centered via flexbox)
```
