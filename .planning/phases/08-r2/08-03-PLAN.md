  <action>
사용자가 다음 인프라 설정을 수행해야 합니다. Claude가 CLI/API로 수행할 수 없는 외부 서비스 설정입니다.

**Step 1: Cloudflare R2 버킷 생성**
1. Cloudflare Dashboard (dash.cloudflare.com) -> R2 Object Storage
2. "Create bucket" 클릭
3. Bucket name: `grapit-assets` (또는 원하는 이름)
4. Location: Automatic (또는 Asia Pacific)
5. "Create bucket" 확인

**Step 2: R2 공개 접근 활성화 (D-07: 읽기 공개)**
1. 생성된 버킷 -> Settings 탭
2. "Public Access" 섹션 -> "Allow Access" 활성화
3. r2.dev subdomain이 생성됨 (예: `pub-xxxxxxxxxxxx.r2.dev`)
4. 이 URL이 `R2_PUBLIC_URL`이 됨 (예: `https://pub-xxxxxxxxxxxx.r2.dev`)

**Step 3: R2 CORS 설정 (D-05, D-06)**
1. 같은 Settings 탭 -> "CORS Policy" 섹션
2. "Add CORS policy" 또는 JSON 편집으로 다음 설정:
```json
[
  {
    "AllowedOrigins": [
      "https://grapit-web-d3c6wrfdbq-du.a.run.app"
    ],
    "AllowedMethods": ["GET", "PUT", "HEAD"],
    "AllowedHeaders": ["content-type"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3600
  }
]
```
- AllowedOrigins: 프로덕션 웹 URL (D-05)
- AllowedHeaders: content-type 명시적 지정 (D-06, 와일드카드 * 불가)
- AllowedMethods: GET(읽기), PUT(presigned 업로드), HEAD(상태 확인)

**Step 4: R2 API 토큰 발급 (R2-01)**
1. Cloudflare Dashboard -> R2 -> Overview (메인 R2 페이지)
2. "Manage R2 API Tokens" 클릭
3. "Create API Token"
4. Permissions: "Object Read & Write"
5. TTL: No expiration (또는 원하는 만료일)
6. Bucket scope: 특정 버킷 (`grapit-assets`)만 허용 (보안 최소 권한)
7. "Create API Token" -> Access Key ID와 Secret Access Key를 기록

**Step 5: 환경변수 값 확인**
아래 6개 값을 기록:
- `R2_ACCOUNT_ID`: Dashboard 우측 Account ID
- `R2_BUCKET_NAME`: `grapit-assets`
- `R2_PUBLIC_URL`: `https://pub-xxxxxxxxxxxx.r2.dev` (Step 2에서 생성된 URL)
- `R2_ACCESS_KEY_ID`: Step 4에서 발급된 Access Key ID
- `R2_SECRET_ACCESS_KEY`: Step 4에서 발급된 Secret Access Key
- `R2_PUBLIC_HOSTNAME`: R2_PUBLIC_URL에서 protocol 제외 (예: `pub-xxxxxxxxxxxx.r2.dev`)

**Step 6: GCP Secret Manager에 시크릿 저장**
GCP Console -> Secret Manager에서 아래 시크릿 생성:
```
r2-account-id        -> R2_ACCOUNT_ID 값
r2-bucket-name       -> R2_BUCKET_NAME 값
r2-public-url        -> R2_PUBLIC_URL 값
r2-access-key-id     -> R2_ACCESS_KEY_ID 값
r2-secret-access-key -> R2_SECRET_ACCESS_KEY 값
```
또는 gcloud CLI:
```bash
echo -n "VALUE" | gcloud secrets create r2-account-id --data-file=-
echo -n "VALUE" | gcloud secrets create r2-bucket-name --data-file=-
echo -n "VALUE" | gcloud secrets create r2-public-url --data-file=-
echo -n "VALUE" | gcloud secrets create r2-access-key-id --data-file=-
echo -n "VALUE" | gcloud secrets create r2-secret-access-key --data-file=-
```

**Step 7: GitHub Repository Secret 추가**
GitHub repo -> Settings -> Secrets and variables -> Actions:
- `R2_PUBLIC_HOSTNAME` = `pub-xxxxxxxxxxxx.r2.dev` (hostname만, protocol 제외)

**Step 8: 로컬 .env 업데이트 (개발용)**
모노레포 루트 `.env`에 R2 환경변수 추가:
```
R2_ACCOUNT_ID=your-account-id
R2_BUCKET_NAME=grapit-assets
R2_PUBLIC_URL=https://pub-xxxxxxxxxxxx.r2.dev
R2_ACCESS_KEY_ID=your-access-key-id
R2_SECRET_ACCESS_KEY=your-secret-access-key
NEXT_PUBLIC_R2_HOSTNAME=pub-xxxxxxxxxxxx.r2.dev
```

|----------|-------------|
| Cloudflare Dashboard -> R2 bucket | Cloudflare 계정 인증 기반 관리 |
| GCP Secret Manager -> Cloud Run | IAM 기반 시크릿 참조 |
| Browser -> R2 presigned PUT | 서명 기반 임시 접근, CORS 정책 적용 |
| Public Internet -> R2 r2.dev | 공개 읽기 접근 (D-07 의도적 설계) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-07 | Spoofing | R2 API Token | mitigate | 버킷 단위 스코프 제한 (grapit-assets만 접근). GCP Secret Manager에 저장, 평문 노출 없음 |
| T-08-08 | Denial of Service | r2.dev rate limiting | accept | D-01에 따라 초기 r2.dev 사용. 트래픽 증가 시 커스텀 도메인 전환 (deferred). 현재 트래픽 수준에서 문제없음 |
