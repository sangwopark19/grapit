## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | 다시 로그인하기 |
| Processing state | 로그인 처리 중... |
| Error: token expired | 로그인 세션이 만료되었습니다. 다시 로그인해주세요. |
| Error: provider unavailable | {provider} 서비스에 일시적으로 연결할 수 없습니다. 잠시 후 다시 시도해주세요. |
| Error: already linked account | 이미 다른 계정에 연결된 소셜 계정입니다. 기존 계정으로 로그인해주세요. |
| Error: generic server | 일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요. |
| Error: network | 네트워크 연결을 확인해주세요. |
| Error: invalid callback | 잘못된 접근입니다. 로그인 페이지에서 다시 시도해주세요. |
| Toast error (existing) | sonner toast.error() -- maintained as-is for non-blocking errors |

Destructive actions: none in this phase (bug fix only, no account deletion or unlinking)

Error UX pattern (from D-08, D-09):
1. Error message displays inline on callback page (replaces loading spinner)
2. Error message includes cause-specific copy from the table above
3. "다시 로그인하기" button appears below error message, routes to /auth
4. For provider-specific errors, {provider} is replaced with "카카오" / "네이버" / "Google"

## Interaction States

### Callback Page Error State

```
+------------------------------------------+
|                                          |
|          [AlertCircle icon 32px]         |
|                                          |
|    로그인 세션이 만료되었습니다.           |
|    다시 로그인해주세요.                    |
|                                          |
|       [ 다시 로그인하기 ]                 |
|                                          |
+------------------------------------------+
```

Layout:
- Centered vertically and horizontally (flex items-center justify-center, same as existing processing state)
- Icon: lucide AlertCircle, 32px (h-8 w-8), color --color-error
- Primary message: 16px / 600 / --color-gray-900, margin-top 16px from icon
- Detail message: 14px / 400 / --color-gray-600, margin-top 8px from primary
- Retry button: shadcn Button default variant, size "lg", full width (max-w-[280px]), margin-top 24px from detail
- Container: max-w-[400px] (matches existing registration flow width)

### Login Form Social Error Display

When redirected back to /auth with ?error= query parameter:
- Display error in existing inline error pattern (same as loginError state)
- Uses text-caption text-error with animate-in fade-in duration-150 (existing pattern from login-form.tsx line 113-115)
- Positioned above the social login buttons section
- Auto-dismiss: none (persists until user takes action)
