<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> public API | Password reset confirm crosses from untrusted browser code to `api.heygrabit.com`. |
| Next.js web runtime -> localhost rewrite | Local rewrite is trusted only for local development and must not become a production routing fallback. |
| Reset token -> backend authority | Browser possesses a token but backend remains the sole authority for token verification and replay rejection. |
| User email -> reset request UX | Password reset request intentionally hides account existence from the browser. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Tampering / Denial of Service | `apps/web/app/auth/reset-password/page.tsx` | mitigate | Use `apiUrl('/api/v1/auth/password-reset/confirm')` and exact URL component test for `https://api.heygrabit.com/api/v1/auth/password-reset/confirm`. |
| T-18-02 | Denial of Service | `apps/web/next.config.ts` | mitigate | Return `[]` rewrites under `NODE_ENV=production` and add `next-config.test.ts` asserting no production `localhost:8080`. |
| T-18-03 | Elevation of Privilege | reset token confirm flow | mitigate | Keep raw fetch and backend `AuthService.resetPassword` authority; do not parse or validate reset tokens in frontend code. |
| T-18-04 | Information Disclosure | reset request mode | mitigate | Preserve `RequestView` catch/finally behavior that always shows success and prevents account enumeration. |
| T-18-05 | Denial of Service | CORS / origin drift | mitigate | Centralize public API origin in `api-url.ts`; Plan 02 documents local/preview/production env contract and smoke evidence. |
</threat_model>
