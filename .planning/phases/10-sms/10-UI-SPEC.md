# Phase 10 — UI Design Contract

> Visual and interaction contract for SMS 인증 실연동 (Twilio mock → Infobip 2FA 실 연동). Generated by gsd-ui-researcher.

> **Phase nature:** Backend-heavy (Infobip SDK 교체, Valkey ThrottlerStorage, hard-fail env). 프론트 UI 영향 범위는 **단일 컴포넌트** `apps/web/components/auth/phone-verification.tsx` 에 한정된다.

## Color

| Role | Value | Phase 10 Usage |
|------|-------|----------------|
| Dominant (60%) | `#FFFFFF` (`--color-background`) | Input 배경, 컴포넌트 wrapper 배경 (호출부 상속) |

| Accent (10%) | `#6C3CE0` (`--color-primary`) | "인증번호 발송" / "확인" 버튼 default variant, focus ring |
| Destructive | `#C62828` (`--color-destructive` / `--color-error`) | 에러 메시지 텍스트, 만료 타이머 텍스트 |

**Accent (10%) reserved for:**
- "인증번호 발송" 버튼 default variant (초기 상태)
- "확인" 버튼 default variant (인증번호 입력 후 활성)
- Input focus-visible ring (`focus-visible:ring-primary`)

**Accent에 넣으면 안 되는 것 (명문화):**
- 쿨다운 중인 재발송 버튼 — accent 색 사용 금지 (disabled outline variant 사용)
- 만료 후 재발송 버튼 — outline variant 유지 (현재 코드 `variant={codeSent && !isExpired ? 'outline' : 'default'}` 를 **모든 재발송 상태에 outline 적용**하도록 교정)

**Destructive vs Warning vs Info 분기 (Phase 10 에러 메시지):**

| 에러 상태 | 색 토큰 | 근거 |
|----------|--------|------|
| 인증번호 만료 (`410/422`) | `text-error` (#C62828) | 재발송 필요 — 사용자 행동 촉구 |
| 인증번호 불일치 (`400`) | `text-error` (#C62828) | 사용자 입력 오류 |
| 재발송 쿨다운 (`429`) | `text-error` (#C62828) | 429 도 destructive로 통일 (D-20은 색 지정 안 함, Phase 9/8 패턴 계승) |
| 국가 미지원 (`+86` 차단) | `text-error` (#C62828) | 가입 불가 상태, 가장 강한 신호 필요 |
| 네트워크 실패 | `text-error` (#C62828) | 기존 toast는 `toast.error` |
| 국가 감지 안내 (정보) | `text-gray-500` | warning 아님, 단순 안내 (`+66 Thailand` 자동 감지 등) |

Phase 10에서 **warning (주황) / info (보라) 배경 surface 블록은 사용하지 않는다**. 모든 상태 메시지는 인라인 텍스트 1줄로 처리.

## Copywriting Contract

### 인풋 Placeholder

| 상태 | Placeholder |
|------|-------------|
| 전화번호 입력 (기본) | `"010-0000-0000 또는 국가코드 포함 +66..."` |
| 전화번호 입력 (국제 모드 감지 시) | `"+66 8 1234 5678"` (감지된 국가 샘플 포맷) — Discretion |
| 인증번호 입력 | `"인증번호 6자리"` (기존 유지) |

**포맷 정책 (국제 번호 대응, Discretion 구체화):**
- 한국 번호(`010`, `011` 등으로 시작)만 `formatPhoneInput` 으로 `010-XXXX-XXXX` 마스킹 유지 (기존 동작)
- 한국 prefix 가 아닌 입력(`+`로 시작하거나 국가코드 2자리 이상 감지)은 마스킹 없이 **raw passthrough** (공백·하이픈만 제거)
- 국가 감지는 planner 재량으로 `libphonenumber-js` (권장) 또는 단순 prefix 매칭. UI에서는 감지 결과만 반영하며, **감지 실패해도 발송 시도는 허용**한다 (사용자가 직접 입력한 번호 신뢰 + 백엔드/Infobip에서 최종 검증)

### Button Labels — 4가지 상태 × 단일 버튼

재발송/발송 버튼은 **하나의 Button** 인스턴스에서 4가지 상태로 렌더링된다:

| 상태 | 조건 | Label | Variant | Disabled |
|------|------|-------|---------|----------|
| `initial` | 아직 발송 안 함 (`!codeSent`) | `인증번호 발송` | `default` | `phone.length < 10 && !isInternational` 이면 true |
| `sending` | 발송 API 호출 중 (`isSending`) | `<Loader2 /> 발송 중...` | `default` | true |
| `cooldown` | 발송 완료 + 30초 이내 (`codeSent && resendCooldown > 0`) | `재발송 (28s)` (초 숫자 동적) | `outline` | true |
| `resend-ready` | 쿨다운 종료 + (기간 내 or 만료 후) (`codeSent && resendCooldown === 0`) | `재발송` | `outline` | false |

**Label 포맷 세부:**
- 쿨다운 숫자는 **정수 초**. 30초부터 내려와 1초까지 매초 업데이트. `재발송 (1s)` → `재발송 (0s)` 렌더 없이 바로 `재발송` 활성으로 전환.
- 숫자 옆 단위는 **`s`** (소문자) — 기존 globals.css/프로젝트에 시간 단위 컨벤션 없음 → `s` 로 통일. `초` 한글 사용 금지 (재발송 라벨 7자 제약 + 버튼 폭 안정성).
- `발송 중` 상태의 Loader2 는 `h-4 w-4 animate-spin` (기존 유지).

### Verify 버튼 Labels

| 상태 | 조건 | Label | Variant | Disabled |
|------|------|-------|---------|----------|
| 기본 | 코드 길이 6 미만 | `확인` | `default` | true |
| 활성 | `code.length === 6 && !isVerifying && !isExpired` | `확인` | `default` | false |
| 검증 중 | `isVerifying` | `<Loader2 />` (아이콘만) | `default` | true |

### Timer Display

만료 타이머(3분)는 **코드 입력 row 아래 독립 row**로 표시된다. 쿨다운(30초)은 버튼 라벨 내부 숫자로만 표시 — **별도 row 렌더링 금지**.

| 상태 | 렌더링 |
|------|--------|
| `timeLeft > 0 && !isVerified` | `<span class="text-caption text-error">{MM:SS}</span>` (예: `02:59`) |
| `timeLeft === 0 && !isVerified` (만료) | `<span class="text-caption text-error">시간 만료</span>` |
| `isVerified` | 타이머 숨김, `CheckCircle2 + "인증 완료"` 로 대체 |

**현재 코드와 차이:** 현재 `phone-verification.tsx:199-207` 은 `text-sm text-error` 를 쓰고 있음. Phase 10 에서 `text-caption text-error` 로 변경하여 다른 에러 메시지들과 토큰 통일.

### 에러 메시지 카피 (D-20 고정, HTTP 상태별 분기)

| HTTP 상태 | 카피 | 표시 위치 | 색 |
|----------|------|----------|-----|
| `400 Bad Request` (잘못된 번호 형식 + 중국 본토) | D-03 원문: `"현재 중국 본토 SMS 인증은 지원되지 않습니다. 다른 국가 번호로 가입해 주세요"` | 인풋 row **아래** 인라인 | `text-caption text-error` |
| `400 Bad Request` (일반 invalid code) | `"인증번호가 일치하지 않습니다"` | 인증번호 row 아래 인라인 | `text-caption text-error` |
| `410/422` (expired or max attempts) | `"인증번호가 만료되었습니다. 재발송해주세요"` | 인증번호 row 아래 인라인 | `text-caption text-error` |
| `429` (resend cooldown 서버 측) | `"잠시 후 다시 시도해주세요"` | 인풋 row 아래 인라인 | `text-caption text-error` |

## Interaction States

### State Machine — 재발송 버튼

```
┌──────────┐  사용자가 phone 입력 + 클릭
│ initial  │─────────────────────────────────▶ sending
└──────────┘                                        │
                                                    │ API success
                                                    ▼
                                               ┌──────────┐
                                   ┌───────────│ cooldown │ (30s, 버튼 outline + disabled, 라벨 "재발송 (Ns)")
                                   │           └──────────┘
                     30s 카운트 0        │ 사용자가 코드 입력 → 검증
                                   ▼
                              ┌──────────────┐
                              │ resend-ready │ (outline + enabled, 라벨 "재발송")
                              └──────────────┘
                                   │ 사용자가 재발송 클릭
                                   ▼
                               sending → cooldown (30s 재시작)
```

**만료 타이머(3분)와 쿨다운 타이머(30초)는 독립 타이머:**
- `timeLeft` (만료) — `useEffect` 기반 1초 단위 감소, 0되면 만료 UI
- `resendCooldown` (쿨다운) — **별도 `useEffect`**, 1초 단위 감소, 0되면 재발송 버튼 활성
- 두 타이머는 발송 성공 시 **동시 재시작** (만료=180s, 쿨다운=30s)
- 재발송 성공 시 **둘 다 재시작**

**레이아웃 공존 규칙:**
- 쿨다운 숫자는 **버튼 라벨 내부에만** 표시 (별도 row 없음)
- 만료 타이머는 **코드 입력 row 아래 전용 row** 에 표시
- 두 숫자가 동시에 보일 수 있음 (발송 직후 0~30s 구간): 만료 `02:59` + 버튼 `재발송 (28s)` — 이는 **의도된 동시 노출**이며 혼란 없음 (각자 다른 의미: 코드 유효기간 vs 재발송 가능 시점)

### State Transitions — 전체 컴포넌트

```
┌─ idle (codeSent=false, isVerified=false) ────────────────────────────┐
│   인풋 + "인증번호 발송" 버튼만 표시                                   │
└──────────────────────────────────────────────────────────────────────┘
                           │ 사용자: 전화번호 입력 + 클릭
                           ▼
┌─ sending (isSending=true) ───────────────────────────────────────────┐
│   버튼 "<Loader2> 발송 중..." (disabled)                              │
└──────────────────────────────────────────────────────────────────────┘
                           │ API 2xx
                           ▼
┌─ code-input (codeSent=true, isVerified=false, timeLeft > 0) ─────────┐
│   인증번호 인풋 + "확인" 버튼                                         │
│   + 만료 타이머 row ("02:59")                                         │
│   + 재발송 버튼 (cooldown 중이면 "재발송 (28s)" outline disabled)     │
└──────────────────────────────────────────────────────────────────────┘
       │                       │ timeLeft === 0                  │
       │ code=6 + 확인 클릭     │                                 │
       ▼                       ▼                                 │
┌─ verifying ──┐     ┌─ expired (timeLeft=0, isVerified=false) ┐ │
│ 확인 버튼     │     │ 인증번호 인풋 disabled                   │ │
│ <Loader2>    │     │ 만료 타이머 row "시간 만료"               │ │
│ (disabled)   │     │ 재발송 버튼 "재발송" outline enabled      │ │
└──────────────┘     └──────────────────────────────────────────┘ │
       │                      │ 재발송 클릭                        │
       │ verify 2xx           └────────────────────┐               │
       ▼                                           │               │
┌─ verified (isVerified=true) ────────────────────────────────────────┐
│   인풋 disabled, 체크 아이콘 + "인증 완료" 만 표시                     │
└─────────────────────────────────────────────────────────────────────┘
       │
       └── onVerified(code) 부모에 전달 → signup-step3 submit
```

**검증 실패 케이스 분기 (D-20):**
```
verifying ─── 400 invalid ────▶ "인증번호가 일치하지 않습니다" 표시 → code-input 복귀
            └─ 410/422 ────────▶ "인증번호가 만료되었습니다. 재발송해주세요" 표시 → expired 진입
            └─ 429 ────────────▶ "잠시 후 다시 시도해주세요" 표시 → code-input 유지
            └─ 5xx/네트워크 ───▶ "일시적인 오류가 발생했습니다..." 표시 → code-input 유지
```

### Input Row 레이아웃

```
┌─────────────────────────────────────────────┬────────────────────┐
│ Input (flex-1)                              │ Button (shrink-0)  │
│  placeholder: 010-0000-0000 또는 국가코드... │  h-9 (default)     │
│  h-11 (from globals)                         │                    │
└─────────────────────────────────────────────┴────────────────────┘
          ↑ gap-2 (8px)
```

**정렬 보정 (현재 코드 이슈):** Input `h-11` (44px) + Button `h-9` (36px default) 높이 차이 발생 → Phase 10 에서 **Button size="default" 유지하되 `size="lg"` (`h-10`, 40px) 전환 권장**. 또는 Input 을 `h-9` 로 낮추는 대안도 가능하나 기존 터치 타겟 44px 원칙 유지 권고 → **`size="lg"` 채택**.

**변경 범위:** 기존 `<Button size="default">` 2개(`발송/재발송`, `확인`) 를 모두 `size="lg"` 로 상향.

## Responsive Behavior

Phase 10 컴포넌트는 `apps/web/app/auth/signup` 과 `apps/web/app/auth/callback` 두 페이지 모두에서 **max-w-[400px]** 컨테이너 내부에 렌더링된다.

| Breakpoint | 동작 |
|------------|------|
| Mobile (< 640px) | Input + Button 가로 배치 유지 (`flex gap-2`). Button `size="lg"` (40px height) 는 모바일 최소 터치 타겟 44px 미달 — 모바일 한정 `size="default"` 44px 유지할지 planner 판단. 현재 iOS 권장 44px 이므로 `h-11` 입력과 동일 높이 유지 결정 시 `size="lg"` 변경 대신 커스텀 클래스 `h-11` 부여도 가능 |
| Tablet (640-1024px) | Desktop 동일 |
| Desktop (≥ 1024px) | 컨테이너 max-w-[400px] 제한으로 레이아웃 불변 |

**한국어 문구 wrap 규칙:**

const RESEND_COOLDOWN_SECONDS = 30;  // D-11 과 동일한 서버 쿨다운

const [resendCooldown, setResendCooldown] = useState(0);
const cooldownTimerRef = useRef<ReturnType<typeof setInterval> | null>(null);

  if (resendCooldown <= 0) {

### 에러 카피 분기 (신규)

```typescript
function mapErrorToCopy(err: unknown): string {
  if (err instanceof ApiError) {
    switch (err.status) {
      case 429:
        return '잠시 후 다시 시도해주세요';
      case 410:
      case 422:
        return '인증번호가 만료되었습니다. 재발송해주세요';
      case 400:
        // 중국 본토 차단 메시지는 서버가 body.message 로 내려줌 (D-03)
        if (err.body?.message?.includes('중국 본토')) return err.body.message;
        return '인증번호가 일치하지 않습니다';
      default:
        return '일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요.';
    }
  }
  return '일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요.';
}
```

### 국가 감지 구현 권장 (Discretion)

```typescript
// apps/web/lib/phone.ts (신규, 권장)
import { parsePhoneNumberFromString } from 'libphonenumber-js/min';

export function detectPhoneLocale(input: string): {
  isKorean: boolean;
  country: string | null;
  e164: string | null;
} {
  if (/^01[016789]/.test(input.replace(/\D/g, ''))) {
    return { isKorean: true, country: 'KR', e164: null };
  }
  const parsed = parsePhoneNumberFromString(input.startsWith('+') ? input : `+${input}`);
  if (parsed?.isValid()) {
    return { isKorean: parsed.country === 'KR', country: parsed.country ?? null, e164: parsed.format('E.164') };
  }
  return { isKorean: false, country: null, e164: null };
}
```

국가명 표시용 매핑 테이블은 planner 재량 (간단하게 `KR: '한국', TH: '태국'` 등 5-10개). 모르는 국가는 `"국제 번호로 SMS를 발송합니다"` fallback.
