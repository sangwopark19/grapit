### DNS 이전 완료 후 순서

DNS 이전이 끝났다는 신호를 받으면 다음 순서로 재개:

1. **선택적으로 먼저**: Sentry 대시보드 → 기존 `grapit-api` / `grapit-web` 프로젝트 → Settings → Slug 필드를 `grabit-api` / `grabit-web`으로 수정 (2클릭, DSN 불변, Secret Manager/GH secret 갱신 불필요 — D-02 exception)
2. **후이즈 대시보드에서** `api.heygrabit.com` CNAME 레코드 추가:
   - 호스트: `api`
   - 타입: `CNAME`
   - 값: `ghs.googlehosted.com`
   - TTL: 3600
   - Proxy: OFF (해당 시)
   - 검증: `dig @8.8.8.8 api.heygrabit.com CNAME +short` → `ghs.googlehosted.com.` 반환 (전파 최대 5분)
3. **GitHub variable 갱신**:
   - `gh variable set CLOUD_RUN_API_URL --body "https://api.heygrabit.com"`
   - (CLOUD_RUN_WEB_URL는 Wave 4 cutover 시 변경)
4. **재개 명령어**: `/gsd-execute-phase 13 --wave 3`
   - 이전 continuation agent가 STEP C (main merge) ~ STEP D (D-13 second deploy, D-12 Sentry event ID 검증) 이어서 진행
   - 성공 시 SUMMARY.md 작성 + STATE/ROADMAP 업데이트
