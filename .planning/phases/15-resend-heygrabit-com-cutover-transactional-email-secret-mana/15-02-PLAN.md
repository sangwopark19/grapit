<objective>
Resend 대시보드에서 `heygrabit.com` 도메인을 추가하고, Resend 가 발급한 DNS 레코드 (SPF MX + SPF TXT + DKIM TXT + DMARC TXT) 를 후이즈 DNS 콘솔에 등록한 뒤, Resend 대시보드의 heygrabit.com 상태가 `Verified` 로 전환될 때까지 대기하고 `dig` 으로 전파를 확증한다. Plan 03 의 Secret rotation 은 이 plan 이 완료(= Verified 상태) 되기 전에는 실행하지 않는다 (D-08 sequence invariant — Resend Verified 먼저, Secret 교체 나중).

Purpose: Wave 2 의 HARD GATE. Resend Verified 가 확보되지 않은 상태로 Secret 을 교체하면 Resend 가 422 domain-not-verified 로 거절하여 silent failure 가 폭주하고 운영 메일 경로가 마비된다 (RESEARCH §Pitfall 2, debug doc 시나리오 B).

Output: (1) Resend 대시보드에 heygrabit.com Verified, (2) 후이즈 DNS 에 Resend 발급 레코드 등록, (3) 15-HUMAN-UAT.md 파일 생성 + 본 plan 범위의 Operational Audit Log 섹션 fill-in.

**중요:** CONTEXT D-03 의 `include:_spf.resend.com` 과 D-05 의 CNAME 가정값은 **참고용**이다. 실제 등록값은 Resend 대시보드 Add Domain 화면이 발급하는 현재 값 (§RESEARCH Pitfall 1, R-01 — Resend 는 AWS SES 기반으로 전환했고 SPF TXT `include:amazonses.com`, DKIM TXT (CNAME 아님) 를 발급하는 것이 2026년 현재 형식) 을 그대로 사용한다. 리터럴 하드코딩 금지.
