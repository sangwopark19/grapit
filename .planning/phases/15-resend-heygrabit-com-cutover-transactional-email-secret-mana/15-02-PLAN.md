<objective>
Resend 대시보드에서 `heygrabit.com` 도메인을 추가하고, **Resend 대시보드 detail page 가 표시하는 DNS record table 의 모든 row (TYPE / NAME / VALUE / PRIORITY) 를 소스 오브 트루스로** 후이즈 DNS 콘솔에 그대로 등록한 뒤, 각 row 를 `dig +short <TYPE> <NAME>` 로 검증하고 Resend 대시보드의 heygrabit.com 상태가 `Verified` 로 전환될 때까지 대기한다. Plan 03 의 Secret rotation 은 이 plan 이 완료(= Verified 상태) 되기 전에는 실행하지 않는다 (D-08 sequence invariant — Resend Verified 먼저, Secret 교체 나중).

Purpose: Wave 2 의 HARD GATE. Resend Verified 가 확보되지 않은 상태로 Secret 을 교체하면 Resend 가 422 domain-not-verified 로 거절하여 silent failure 가 폭주하고 운영 메일 경로가 마비된다 (RESEARCH §Pitfall 2, debug doc 시나리오 B).

Output: (1) Resend 대시보드에 heygrabit.com Verified, (2) 후이즈 DNS 에 Resend 대시보드 row 전체 그대로 등록, (3) 15-HUMAN-UAT.md 파일이 Task 0 에서 shell 로 생성된 뒤 Task 1~3 이 audit trail 을 in-line 기록.

**중요 (REVIEWS MEDIUM M1 반영):** CONTEXT D-03 의 `include:_spf.resend.com`, D-05 의 `resend._domainkey` CNAME 가정값은 **참고용에 불과하다**. 실제 등록값은 Resend 대시보드 Add Domain 화면 detail page 가 발급하는 현재 값 (type/name/value/priority 전부) 을 그대로 사용한다 — Resend 는 시기에 따라 AWS SES 기반 TXT/MX 조합을 발급하기도 하고 CNAME redirection 형태로 DKIM selector 를 위임하기도 한다. **selector 이름, record type, record 개수 모두 고정 가정 금지**. 구 grapit.com 은 이 phase 범위에서 건드리지 않는다 (D-02 — Plan 03 Task 5 에서 제거).
