## Root Cause Analysis (SUMMARY에서 재구성)

계획된 A/B/C 가설 공간 외의 4번째 원인이 발견됨:

1. GitHub repo에 TEST_USER_EMAIL / TEST_USER_PASSWORD secret이 미설정 상태였음
2. GitHub Actions는 unset secret를 빈 문자열("")로 env에 주입함 (표준 동작)
3. `helper`의 `process.env['TEST_USER_EMAIL'] ?? 'admin@grapit.test'` 패턴에서 `??`는 null/undefined만 catch → 빈 문자열은 통과
4. Playwright가 `{"email":"","password":""}` (26 bytes) 전송 → NestJS가 정상 파싱 → passport-local이 빈 credentials reject → 401
5. curl smoke는 하드코딩된 리터럴을 사용하므로 영향 없이 HTTP 200 유지됨

**Fix (D-3 분기):**
- `??` → `||` (빈 문자열도 fallback — D-1 helper hardening)
- `gh secret set TEST_USER_EMAIL` + `gh secret set TEST_USER_PASSWORD` (D-2 GitHub secrets)
- confirm/page.tsx `useRef` latch (Scenario 2/3를 막던 React StrictMode double-fire)
