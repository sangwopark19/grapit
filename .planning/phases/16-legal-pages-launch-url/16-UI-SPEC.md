# Phase 16 — UI Design Contract

> Visual and interaction contract for the "Legal pages launch" phase. Generated by gsd-ui-researcher from `16-CONTEXT.md` decisions D-01~D-15 plus existing globals.css tokens and `terms-markdown.tsx` mappings.
> 본 phase 는 **content-heavy / low-interactivity** 성격. 새 디자인 토큰을 도입하지 않고, 기존 `TermsMarkdown` 매핑(`text-base`/`text-caption`/`text-primary`)을 재사용한다. 핵심 시각 계약은 (1) `apps/web/app/legal/layout.tsx` container, (2) `showH1=true` 일 때의 H1 렌더, (3) Footer 링크 3개의 실 경로 교체, (4) `LegalDraftBanner` 시각 요소 소멸이다.

## Spacing Scale

기존 `globals.css:55-62` 8-point grid 토큰 완전 재사용. **Phase 16 은 신규 spacing 토큰을 도입하지 않는다.**

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | (미사용) |
| sm | 8px | H2 → H3 inline gap (기존 `mt-2` 보존) |
| md | 16px | TermsMarkdown 내부 P/UL 간격 (`mt-2`, `mt-4`) — 기존 컴포넌트 값 |
| lg | 24px | H1 ↔ 본문 첫 H2 간격 (`mt-6`), 좌우 padding 모바일 (`px-4`/`md:px-6`) |
| xl | 32px | Layout container 상하 패딩 (`pt-8`/`pb-12`) — mypage 패턴 차용 |
| 2xl | 48px | Layout container 데스크톱 상하 패딩 (`md:pt-12`/`md:pb-16`), H1 ↔ 본문 사이 데스크톱 spacing (`md:mt-8`) |
| 3xl | 64px | (Phase 16 미사용) |

### Layout container exact values (`apps/web/app/legal/layout.tsx`)

```jsx
<main className="mx-auto w-full max-w-[760px] px-4 pt-8 pb-16 md:px-6 md:pt-12 md:pb-24">
  {children}
</main>
```

| Property | Value | 근거 |
|----------|-------|------|
| Horizontal centering | `mx-auto w-full` | 기존 mypage / genre 패턴 일관 |
| **Max width** | **`max-w-[760px]`** | 본문 가독성 최적치(75자/줄 ≈ 750px @ 16px Pretendard). 1200px 그리드보다 좁힘 — 법적 문서는 reading-optimized container. mypage(`max-w-[600px]`)는 form-friendly, 약관(80~100라인 본문)은 600px 보다 760px 가 줄당 글자 수를 안정시킴. **Token 아닌 절대값**으로 허용 — page-type 별 reading-width 는 spacing scale 과 분리 차원. |
| Mobile padding | `px-4 pt-8 pb-16` | xs(좌우 16px) + xl(상 32px) + 2xl 직전(하 64px) |
| Desktop padding | `md:px-6 md:pt-12 md:pb-24` | lg(좌우 24px) + 2xl(상 48px) + 3xl(하 96px ≈ `pb-24`) |

**예외 및 허용 절대값:**
- Layout container max-width `760px` — token 미정의. Reading-optimized width 로 page 단독 허용 (위 근거 표 참조).
- `TermsMarkdown` 내부의 `mt-1` (`<li>` 항목 간격) — 4px 임. xs token 안에 포함되며 기존 컴포넌트 값. Phase 16 변경 금지.

**금지:**
- 신규 spacing 토큰(`@theme` 추가) 금지.
- Container 내부에 별도 카드/섹션 박스(`bg-white shadow-sm rounded-lg`) 추가 금지 — 본문 자체가 콘텐츠. 시각적 chrome 최소.
- Footer 링크 사이 구분자 spacing 변경 금지 — 기존 `gap-2 | gap-2` 유지.

## Typography

기존 semantic 토큰 + `TermsMarkdown` 컴포넌트 매핑 재사용. **신규 size/weight 도입 금지. 정확히 4 sizes + 2 weights.**

| Role | Size | Weight | Line Height | Where Used in Phase 16 |
|------|------|--------|-------------|------------------------|
| Display (H1) | 28px (`text-display`) | 600 (`font-semibold`) | 1.2 (`leading-[1.2]`) | `TermsMarkdown showH1=true` 시 페이지 H1 (이용약관/개인정보처리방침/마케팅 수신 동의) |
| Heading (H2) | 16px (`text-base`) | 600 (`font-semibold`) | Tailwind default 1.5 | TermsMarkdown 기존 H2 매핑 (`text-base font-semibold text-gray-900`) — 약관 조항 제목 (예: `제1조 (목적)`) |
| Body (H3 / strong) | 14px (`text-caption`) | 600 (`font-semibold`) | Tailwind default 1.5 | TermsMarkdown 기존 H3 매핑 (`text-caption font-semibold text-gray-800`), strong (`font-semibold text-gray-900`) |
| Caption (p / ul / ol) | 14px (`text-caption`) | 400 (default) | `leading-relaxed` (1.625) | TermsMarkdown 기존 p/ul/ol 매핑 (`text-caption leading-relaxed text-gray-700`) — 약관 본문 |

### H1 rendering contract (D-09)

`TermsMarkdown` 의 현 `h1: () => null` 매핑은 dialog 호환을 위해 보존. `showH1?: boolean` prop 추가:

```tsx
// terms-markdown.tsx — 변경 후 매핑 (Phase 16 추가)
const buildComponents = (showH1: boolean): Components => ({
  ...components,
  h1: showH1
    ? (props) => (
        <h1
          className="text-display font-semibold leading-[1.2] text-gray-900"
          {...props}
        />
      )
    : () => null,
});

export function TermsMarkdown({
  children,
  showH1 = false,
}: {
  children: string;
  showH1?: boolean;
}) {
  return (
    <ReactMarkdown components={buildComponents(showH1)} remarkPlugins={[remarkGfm]}>
      {children}
    </ReactMarkdown>
  );
}
```

**H1 spacing 처리:**
- H1 자체는 별도 `mt-*` 부여하지 않음 — Layout container 의 `pt-8`/`md:pt-12` 가 상단 여백 담당.
- H1 → 첫 H2 사이: 기존 H2 매핑의 `mt-6 first:mt-0` 패턴이 자동 적용 — H1 다음에 H2 가 오면 `mt-6` (24px). H1 ↔ H2 사이가 24px (모바일) / 32px (데스크톱) 로 자연스러운 hierarchy break.
- 추가 wrapper 박스 / divider 도입 금지.

**Phase 16 금지:**
- `text-display` (28px) 외의 H1 크기 (예: `text-2xl`/`text-3xl`) 금지.
- TermsMarkdown 내부 매핑(h2/h3/p/ul/ol/li/strong/a/hr) 변경 금지 — dialog 회귀 위험.
- 신규 폰트 스케일(`text-lg`/`text-xl`/`text-2xl`) 도입 금지.
- `font-medium`(500) / `font-bold`(700) 사용 금지 — `font-semibold`(600) + default(400) 만.
- 커스텀 `leading-*` 추가 금지 — 기존 `leading-[1.2]`(display) / `leading-relaxed`(body, 1.625) / Tailwind default(1.5) 만.

### Footer 타이포 (변경 없음)

Footer 의 `이용약관 | 개인정보처리방침 | 고객센터` 라인은 기존 `text-sm text-gray-900` 유지. `개인정보처리방침` 만 `font-semibold` (정통망법 강조 의무). Copyright 라인 `text-sm text-gray-500` 유지. **Phase 16 에서 Footer 타이포 변경 금지.**

### Accent (`#6C3CE0`) reserved for (Phase 16):

1. **Inline link in legal markdown** — `TermsMarkdown` 의 `a` 매핑(`text-primary underline hover:text-primary/80`). 약관 본문 내 외부 링크(예: 우편 주소·통신판매업 신고번호 안내 링크) 가 등장할 경우 자동 적용. 기존 매핑 그대로 유지.
2. **Footer 링크 hover 시 underline** — 기존 `hover:underline` 유지. `text-primary` 직접 적용은 금지(아래 NOT allowed 참조).
3. **Focus ring** — `--color-ring` 전역 사용 유지. Footer 링크 / Layout main 영역 모두 동일.
4. **Signup-step2 "보기" 버튼 hover** — 기존 `text-gray-500 underline hover:text-primary` 유지 (Phase 16 변경 없음).

### Accent NOT allowed on (Phase 16):

- H1/H2/H3 텍스트 — 모두 `text-gray-900` / `text-gray-800` (기존 `TermsMarkdown` 매핑) 유지.
- Footer 링크 default state — `text-gray-900` 유지. Hover 시 `text-primary` 로 색상 변경 **금지** — `hover:underline` 만 허용 (기존 패턴 유지).
- 본문 P/UL/OL 텍스트 (`text-gray-700` 유지).
- "개인정보처리방침" 강조(`font-semibold`)에 색상 추가 금지 — weight 만으로 강조 (정통망법 요건은 weight 강조로 충족).

### Destructive actions

**Phase 16 에서 destructive action 없음.** 약관 페이지는 read-only. signup-step2 의 동의 체크박스 / Dialog 닫기는 Phase 16 변경 범위 밖 (D-11).

### Copy tone guidelines

- 존댓말 유지 (`~합니다`, `~안내합니다`).
- Page H1 / `<title>` 둘 다 한국어. `<title>` 끝에 `— Grabit` 부착(em-dash). 이는 다른 page 의 `<title>` 패턴(예: root layout 의 `Grabit - 공연 티켓 예매`) 와 dash 종류가 다르지만, em-dash(`—`) 가 한국어·법적 문서 톤에 더 적합 — Phase 16 단독 결정.
- Meta description 은 "~을 안내합니다" 형식으로 통일 (3개 페이지 일관성).

### LegalDraftBanner 카피 소멸 (D-05)

`본 약관은 런칭 전 법률 검토를 거쳐 교체될 초안입니다.` 카피는 컴포넌트 삭제로 함께 사라짐. **dialog · 공개 페이지 양쪽에서 동시에 소멸** — D-05 의 명시적 결정.

### Footer "고객센터" mailto 카피 (D-03)

```html
<a href="mailto:support@heygrabit.com" class="hover:underline">고객센터</a>
```

- `mailto:` 가 안되는 클라이언트(웹메일 미연동 브라우저)에서는 클릭 시 OS 기본 메일 클라이언트 prompt — 별도 fallback UI 없음 (FAQ/CS 페이지는 Deferred).
- `target="_blank"` / `rel` 부착 금지 — `mailto:` 는 새 탭 의미 없음.
- 시각적 hint(메일 icon 등) 추가는 **금지** — 다른 두 링크와 시각 균형 유지.

### 반응형 breakpoint

- Tailwind 기본 `md` = 768px.
- **`< 768px` (모바일):** padding `px-4 pt-8 pb-16` (좌우 16px, 상 32px, 하 64px). MobileTabBar(56px + safe-area) 와 충돌 없음 — `pb-16` (64px) 이 tab bar 보다 먼저 나오는 spacer 역할.
- **`≥ 768px` (데스크톱):** padding `px-6 pt-12 pb-24` (좌우 24px, 상 48px, 하 96px). Footer 가 자동 노출 (`md:block hidden`).
- Container `max-w-[760px]` 는 데스크톱(768px+) 에서 의미 있음. 모바일에서는 `w-full` 이 우선.

### Long-form scroll behavior

- 페이지 내 anchor 링크 자동 생성 **하지 않음** — `react-markdown` 기본은 H2 에 `id` 자동 부여하지 않음. Phase 16 에서 anchor / TOC 추가 금지 (Deferred).
- 약관 본문이 80~100라인이지만 자연 스크롤로 충분 — 별도 sticky nav / scrollspy 도입 금지.
- `html { scrollbar-gutter: stable }` 는 `globals.css:165` 에 이미 적용 — 스크롤바 등장 시 layout shift 없음.

## Accessibility Contract

- **Landmark structure:** `<main>` (legal/layout.tsx) — 1 회. Root layout 에는 `<main>` 없음 확인. Header/Footer 는 root LayoutShell 의 `<GNB>` / `<Footer>` 가 시맨틱 역할(`<header>`/`<footer>` semantics 는 각 컴포넌트 내부 책임 — Phase 16 변경 없음).
- **Heading hierarchy:** `<h1>` (page title, 1 회) → `<h2>` (조항 제목, 다회) → `<h3>` (소제목, 옵션). H1 → H2 사이 H 단계 점프 없음. H1 은 `showH1=true` 일 때만 렌더 — dialog (`showH1=false`) 에서는 DialogTitle (`<h2>`) 이 이미 제목 역할.
- **`lang="ko"` (기존 root layout):** 약관 본문이 한국어인 만큼 root `<html lang="ko">` (apps/web/app/layout.tsx:21) 가 그대로 적용됨. 변경 없음.
- **링크 접근성:**
  - Footer Link 3개 — 텍스트 카피만으로 목적 명확 (`이용약관`, `개인정보처리방침`, `고객센터`). 별도 `aria-label` 불필요.
  - mailto 링크 — `<a href="mailto:support@heygrabit.com">고객센터</a>` 시각 카피로 충분. 시각장애인 사용자에게도 `고객센터` 가 곧 메일 보내기로 인식됨 (스크린 리더가 `mailto:` 자동 announce).

- **Focus indicator:**
  - Tailwind / shadcn 기본 `--color-ring` (`#6C3CE0`) 적용. 모든 포커스 가능 요소(Footer 링크 3, 본문 외부 링크) 에 자동.
  - 신규 focus 스타일 추가 금지 — 기본 ring 사용.
- **Color contrast (WCAG AA Normal):** 모든 텍스트 / 링크가 AA 이상 (위 Color §Contrast check 표 참조 — 최저 4.6:1).
- **Screen reader test target (Plan / HUMAN-UAT):**
  - VoiceOver / NVDA 로 `/legal/terms` 진입 → H1 announce → H2/H3 hierarchy → 본문 P 순서대로 읽힘.
  - `mailto:` 링크 `고객센터` announce 시 `support@heygrabit.com` 자동 발음 (스크린 리더 표준).
