## Color System

### Brand Palette (WCAG AA Compliant)

All foreground colors pass WCAG 2.1 AA (4.5:1 minimum) on both White (#FFFFFF) and Gray-100 (#F5F5F7) backgrounds.

| Role | HEX | Contrast on White | Contrast on Gray-100 | Usage |
|------|------|-------------------|---------------------|-------|
| Primary | #6C3CE0 | 6.25:1 | 5.74:1 | CTA buttons, active links, focus rings, active tabs |
| Gray-900 (text) | #1A1A2E | 17.06:1 | 15.67:1 | Headings, body text, labels |
| Gray-100 (surface) | #F5F5F7 | N/A (bg) | N/A (bg) | Secondary surface, card backgrounds |
| White | #FFFFFF | N/A (bg) | N/A (bg) | Primary surface, page background |

### Semantic Colors (WCAG AA Compliant)

Original values from D-16 adjusted for text accessibility. Each color has a **text** variant (passes AA on white/gray-100) and a **surface** variant (lighter, for backgrounds only).

| Role | Text HEX | Surface HEX | Text on White | Usage |
|------|----------|-------------|---------------|-------|
| Error | #C62828 | #FEF2F2 | 5.62:1 | Validation errors, inline error text, destructive states |
| Success | #15803D | #F0FDF4 | 5.02:1 | Success messages, completion states |
| Warning | #8B6306 | #FFFBEB | 5.41:1 | Caution notices, deadline reminders |
| Info | #6C3CE0 | #F3EFFF | 6.25:1 | Informational notices (same as Primary) |

### Deferred Colors (Phase 2+)

These colors are defined for palette completeness but NOT used in Phase 1 UI. Their accessibility will be re-verified in the relevant phase UI-SPEC.

| Role | Original (D-14) | AA-Compliant | Phase |
|------|-----------------|--------------|-------|
| Secondary (orange) | #FF6B35 | #BF4B18 (4.96:1) | Phase 2 (badges, promotions) |
| Accent (teal) | #00D4AA | #007A62 (5.30:1) | Phase 2+ (highlights) |

### 60/30/10 Distribution (Phase 1)

| Split | Color | Elements |
|-------|-------|----------|
| 60% Dominant | #FFFFFF | Page background, form backgrounds, modal backgrounds |
| 30% Secondary | #F5F5F7 | GNB background, footer background, card surfaces, input backgrounds |
| 10% Accent | #6C3CE0 | Primary CTA buttons, active tab indicators, focus rings, text links, checkbox checked state |

**Accent (#6C3CE0) reserved for:**
1. Primary CTA buttons ("로그인", "회원가입", "다음", "완료")
2. Active tab underline (로그인/회원가입 탭)
3. Focus ring on interactive elements (2px solid)
4. Text links
5. Checkbox/radio checked state
6. Step indicator active state

## Spacing Scale

Declared values (4px base, 8-point grid):

| Token | Value | Usage in Phase 1 |
|-------|-------|-------------------|
| xs | 4px | Icon-text gap (e.g., checkbox label), inline padding |
| sm | 8px | Compact spacing: between form hint text and field, badge padding |
| md | 16px | Default spacing: form field vertical gap, card inner padding, input horizontal padding |
| lg | 24px | Section padding: GNB horizontal padding, form section gaps, footer padding |
| xl | 32px | Layout gaps: between major page sections |
| 2xl | 48px | Major section breaks: hero to content, content to footer |
| 3xl | 64px | Page-level vertical padding (top/bottom page margin) |

Exceptions:
- Input height: 44px (touch target minimum per WCAG, matches NOL ticket pattern)
- Button height large: 48px (primary CTA)
- Button height medium: 40px (secondary actions)
- Button height small: 32px (tertiary actions)
- GNB height: 64px
- Footer minimum height: 120px
- Step indicator dot: 32px diameter

## Typography

Font: **Pretendard** via `next/font/local` with `PretendardVariable.woff2`.
Fallback: `-apple-system, "Malgun Gothic", sans-serif`.
Code/numbers: `ui-monospace, SFMono-Regular, Menlo, monospace`.

### Type Scale (Phase 1 Subset)

Phase 1 uses exactly 4 sizes and 2 weights:

| Role | Size | Weight | Line Height | Letter Spacing | Usage |
|------|------|--------|-------------|----------------|-------|
| Display | 28px | 600 (semibold) | 1.2 | -0.02em | Home hero heading only |
| Heading | 20px | 600 (semibold) | 1.3 | -0.01em | Page titles, section headings, step titles |
| Body | 16px | 400 (regular) | 1.5 | 0 | Body text, form labels, button text, nav items |
| Caption | 14px | 400 (regular) | 1.5 | 0 | Helper text, error messages, footer links, meta info |

### Why These 4 Sizes

- **28px Display:** Home hero "곧 다양한 공연이 찾아옵니다" -- needs visual weight without being oversized
- **20px Heading:** Auth page titles ("로그인", "회원가입"), step headings ("약관 동의"), My Page sections
- **16px Body:** Primary reading size. Form inputs, labels, button text, GNB items. Matches 16px base from UIUX guide
- **14px Caption:** Secondary info. Input validation messages, helper text below fields, footer copyright, step descriptions

### Font Loading

```typescript
// app/fonts.ts
import localFont from 'next/font/local';

export const pretendard = localFont({
  src: './fonts/PretendardVariable.woff2',
  display: 'swap',
  weight: '100 900',
  variable: '--font-pretendard',
});
```

## Page Contracts

### /auth (Login + Signup)

**Layout:** Centered single column, max-width 400px, vertically centered on desktop.

**Tab structure:**
- Tab 1: "로그인"
- Tab 2: "회원가입"
- Active tab: Primary color underline (2px), 600 weight text
- Inactive tab: Gray-500 (#71717A) text, 400 weight

#### Login Tab

```
[Tab: 로그인 | 회원가입]

이메일
[______________________] <- Input 44px
비밀번호
[______________________] <- PasswordInput 44px with eye icon

[      로그인      ] <- Primary Button 48px, full width

[비밀번호 찾기] <- text link, 14px, right-aligned

─────── 또는 ───────  <- Separator with text

[카카오로 시작하기]    <- SocialLoginButton, yellow
[네이버로 시작하기]    <- SocialLoginButton, green
[Google로 시작하기]    <- SocialLoginButton, white+border
```

**Field spacing:** 16px between label and input, 16px between input groups, 24px before primary CTA, 16px between CTA and password-find link, 24px before separator, 12px between social buttons.

#### Signup Tab (3-Step Flow)

**Step Indicator:** 3 dots connected by lines. Active dot: Primary filled. Completed dot: Primary filled + checkmark. Future dot: Gray-200 outline.

**Step 1 -- 이메일/비밀번호:**
```
[Step: (1)----(2)----(3)]

이메일 *
[______________________]
(이메일 형식 검증: blur 시)

비밀번호 *
[______________________] <- eye toggle
(8자 이상, 영문+숫자+특수문자 포함)

비밀번호 확인 *
[______________________] <- eye toggle

[      다음      ] <- Primary Button 48px
```

**Step 2 -- 약관 동의:**
```
[Step: (1)----(2)----(3)]

[x] 전체 동의
─────────────────────
[x] 이용약관 동의 (필수)          [보기] <- text link
[x] 개인정보처리방침 동의 (필수)    [보기] <- text link
[ ] 마케팅 수신 동의 (선택)        [보기] <- text link

[      다음      ] <- Primary Button, disabled until required checked
```

Checkbox size: 20px. Gap between checkbox and label: 8px. "보기" opens Dialog with full terms text.

**Step 3 -- 추가 정보:**
```

[______________________]

[YYYY]  [MM]  [DD]   <- 3 separate inputs, inline

(03:00 카운트다운 타이머)

[      가입 완료      ] <- Primary Button, disabled until all valid + phone verified
```

## Interaction Patterns

### Form Validation

- **Trigger:** On blur (first interaction), then on change (after first error)
- **Error display:** 14px #C62828 text below the field. Input border changes to #C62828.
- **Required indicator:** Red asterisk (*) after label text
- **Submit with errors:** Scroll to first error field + focus

### Loading States

| Action | Loading Pattern |
|--------|----------------|
| Login submit | Button text changes to spinner + "로그인 중..." Button disabled |
| Signup submit | Button text changes to spinner + "처리 중..." Button disabled |
| Social login click | Button shows spinner. Redirect to OAuth provider |
| Phone verify send | Button shows spinner + "발송 중..." for 2s |
| Profile save | Button shows spinner + "저장 중..." Button disabled |
| Page navigation | Top progress bar (NProgress style, Primary color) |

### Transitions

- Page transitions: none (standard Next.js navigation)
- Tab switch (login/signup): content fade, 150ms ease-in-out
- Step transition (signup): slide-left, 200ms ease-out
- Error message appear: fade-in, 150ms
- Toast: slide-in from top, 300ms. Auto-dismiss 3s (success) or 5s (error)

### Phone Verification Timer

- Starts at 03:00 (3 minutes) on send
- Displays as MM:SS countdown, 14px, #C62828 text
- On expire: timer text changes to "시간 만료". Verify button disabled. "재발송" button appears
- On successful verify: timer disappears. Field shows green checkmark icon + "인증 완료" in #15803D
