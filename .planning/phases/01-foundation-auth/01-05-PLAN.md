    **Zustand auth store (apps/web/stores/use-auth-store.ts):**
    'use client' (Zustand is client-only per Research Pitfall 6).
    ```typescript
    import { create } from 'zustand';
    import { UserProfile } from '@grapit/shared';

    interface AuthState {
      accessToken: string | null;
      user: UserProfile | null;
      isInitialized: boolean; // false until first refresh attempt completes
      setAuth: (accessToken: string, user: UserProfile) => void;
      clearAuth: () => void;
      setInitialized: () => void;
    }

    export const useAuthStore = create<AuthState>((set) => ({
      accessToken: null,
      user: null,
      isInitialized: false,
      setAuth: (accessToken, user) => set({ accessToken, user, isInitialized: true }),
      clearAuth: () => set({ accessToken: null, user: null }),
      setInitialized: () => set({ isInitialized: true }),
    }));
    ```
    Access token in memory only (NOT localStorage -- per Research anti-pattern). Refresh token is httpOnly cookie managed by browser.

    **API client (apps/web/lib/api-client.ts):**
    Create a fetch wrapper that:
    1. Prepends API_URL (process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8080')
    2. Adds Authorization: Bearer {accessToken} from useAuthStore.getState()
    3. Always sends credentials: 'include' (for httpOnly cookie -- per Research Pitfall 3)
    4. On 401 response: attempts silent refresh via POST /api/v1/auth/refresh (credentials: 'include')
       - If refresh succeeds: update store with new accessToken, retry original request
       - If refresh fails: clear auth store, redirect to /auth
    5. Implements request deduplication for refresh: use a module-level promise variable so concurrent 401s share one refresh call (per Research Pitfall 2)
    6. Content-Type: application/json for all requests
    7. Returns typed response via generics: apiClient.post<AuthResponse>('/api/v1/auth/login', body)

    Methods: get<T>(path), post<T>(path, body), patch<T>(path, body), delete<T>(path)

    **Auth helpers (apps/web/lib/auth.ts):**
    ```typescript
    export async function initializeAuth(): Promise<void> {
      // Called once on app load in root layout
      // Attempt silent refresh to restore session
      try {
        const res = await fetch(`${API_URL}/api/v1/auth/refresh`, {
          method: 'POST',
          credentials: 'include',
        });
        if (res.ok) {
          const data = await res.json();
          // Fetch user profile with new token
          const userRes = await fetch(`${API_URL}/api/v1/users/me`, {
            headers: { Authorization: `Bearer ${data.accessToken}` },
            credentials: 'include',
          });
          if (userRes.ok) {
            const user = await userRes.json();
            useAuthStore.getState().setAuth(data.accessToken, user);
            return;
          }
        }
      } catch {
        // No valid session, that's fine
      }
      useAuthStore.getState().setInitialized();
    }
    ```
