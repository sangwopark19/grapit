## Typography

Must reuse semantic tokens declared at `globals.css:57-61` plus the 3 body-scale Tailwind utilities already in use across admin pages. Declared set for this phase — exactly 4 sizes, exactly 2 weights (regular 400 + semibold 600).

| Role | Size | Weight | Line Height | Where Used |
|------|------|--------|-------------|------------|
| Page title | 20px (`text-xl`) | 600 (`font-semibold`) | 1.2 (`leading-tight` implicit) | `<h1>대시보드</h1>` at top of `/admin/page.tsx` (matches `admin-booking-dashboard.tsx:74`) |
| KPI value | 20px (`text-xl`) | 600 (`font-semibold`) | 1.2 | `AdminStatCard` big number (existing `admin-stat-card.tsx:32`) |
| Section label / KPI label / body | 14px (`text-sm`) | 400 | 1.5 | KPI card labels, chart axis labels, table body cells |
| Caption / muted copy | 12px (`text-xs`) | 400 | 1.5 | Legend text, "기간: 최근 30일" subcopy, empty-state body |

Additional fonts/weights beyond this table are NOT allowed in phase 11 components. Specifically:
- No `text-2xl` / `text-3xl` / `text-base` / `text-lg` in dashboard views.
- No `font-medium` / `font-bold` — use 400 or 600 only.

Source-of-truth mapping:
- `text-xl` + `font-semibold` = existing admin page-title convention (`text-xl font-semibold text-gray-900`).
- `text-sm` + default weight = existing label/body convention (`text-sm text-gray-600`).
- `text-xs` = reserved for chart legend / muted caption only.

Line heights rely on Tailwind defaults; do not override with custom `leading-*` except where a pre-existing component sets it.

## Color

60/30/10 split derived from existing tokens. All values already defined in `globals.css:6-26`.

| Role | Value | Token | Usage in this phase |
|------|-------|-------|---------------------|
| Dominant (60%) | `#FFFFFF` | `--color-background` / `bg-white` | Page background, KPI card surface, chart panel surface, table surface |
| Secondary (30%) | `#F5F5F7` | `--color-gray-100` / `--color-secondary` / `bg-gray-100` | Sidebar (`apps/web/components/admin/admin-sidebar.tsx:30`), table header row, chart tooltip background, active tab hover |
| Accent (10%) | `#6C3CE0` | `--color-primary` | **Reserved-for list below** |
| Destructive | `#C62828` | `--color-destructive` / `--color-error` | KPI card "취소율" icon tint only when percent ≥ 10%; otherwise NOT used in dashboard (no destructive actions on this page) |

Accent (`#6C3CE0` / `primary`) reserved for:
1. Active sidebar item ("대시보드") left border + background tint (existing pattern: `border-l-[3px] border-primary bg-primary/5 text-primary`).
2. Area-chart primary series stroke + gradient fill (매출 추이 area chart = revenue line).
3. Active state of the 7/30/90-day ToggleGroup button (selected segment).
4. Focus ring on interactive elements (`--color-ring`, already globally wired).
5. Top 10 ranking row #1 rank badge background (optional — if planner chooses to emphasize rank 1).

Accent is NOT allowed on:
- Body text, headings, labels (keep `text-gray-900` / `text-gray-600`).
- KPI numbers (all KPI values are `text-gray-900`, NOT purple).
- Inactive controls, idle chart series after the primary one.

### Chart color palette (shadcn Chart `--chart-1~5` CSS variables)

`npx shadcn add chart` will add `--chart-1~5` to `:root` in `globals.css`. Override the defaults to match Grapit brand:

| Token | Hex | Assigned series |
|-------|-----|-----------------|
| `--chart-1` | `#6C3CE0` (primary) | Area chart primary series (매출), donut primary slice (largest genre), bar chart primary series (가장 많이 쓰인 결제수단) |
| `--chart-2` | `#8B6DE8` (primary-lighter, 500-ish) | donut 2nd slice, bar 2nd series |
| `--chart-3` | `#B8A3EF` (primary-300) | donut 3rd slice, bar 3rd series |
| `--chart-4` | `#D1D1DB` (`--color-gray-300`) | donut 4th slice, bar 4th series |
| `--chart-5` | `#A1A1AA` (`--color-gray-400`) | donut "기타" / bar 5th series |

This keeps the dashboard monochromatic (purple gradient + gray) instead of a rainbow palette — aligned with the existing brand tone. No `--chart-1` through `--chart-5` values other than the five above may be declared.

Exception: if a single-series chart uses only one color, use `--chart-1` and omit the rest.

**None in this phase.** Dashboard is read-only; no delete, no refund, no bulk actions. Confirmation dialogs and `AlertDialog` components are NOT required for `/admin/page.tsx`. (Existing refund flow in `/admin/bookings` is out-of-scope for phase 11.)

### Copy tone guidelines

- 존댓말 (formal polite) — matches existing admin copy ("환불이 완료되었습니다", "예매 내역이 없습니다").

## Interaction & State Contract

| State | Behavior |
|-------|----------|
| Initial load | Skeleton placeholders for each section (KPI row + 3 chart panels + Top 10 rows). No flash of empty content. |
| Period change (7/30/90) | All 3 charts (revenue, genre, payment) refetch simultaneously. Top 10 does NOT refetch (fixed 30d per D-10). UI shows skeleton over the 3 chart panels only; KPI row + Top 10 remain stable. |
| Refetch on focus | TanStack Query default `refetchOnWindowFocus: true` acceptable — 60s TTL on server means hit rate high. |
| Error (any endpoint) | Per-section error boundary: failed sections show error copy + `다시 시도` button. Other sections remain functional. Do NOT blank the whole page on a single endpoint failure. |
| Empty data | Per-section empty copy per Copywriting Contract above. |
| Hover on chart element | `ChartTooltip` (shadcn) with `text-xs` label + `text-sm` value. Tooltip background `bg-white shadow-sm border`, foreground `text-gray-900`. |
| Focus / keyboard | Period filter toggles accessible via Tab + arrow keys (ToggleGroup default). Chart elements not individually focusable (recharts default). |
| `prefers-reduced-motion` | Disable chart enter animations when set. Use `animationDuration={0}` on recharts components conditionally. |

## Accessibility Contract

- All KPI cards and chart sections use semantic `<section aria-labelledby="...">` wrapper with an `<h2>` section heading (visually `text-sm font-semibold text-gray-600 mb-2`).
- Period filter: `<ToggleGroup aria-label="기간 선택">`.
- Each chart has an off-screen `<caption>`-style summary for screen readers describing the chart type and current data (e.g., `매출 추이 area chart, 최근 30일, 최고 매출 일자: 2026-04-15`). Implement as visually-hidden `<p className="sr-only">`.
- Color contrast: all text uses `text-gray-900` (on white ≥ 15:1) or `text-gray-600` (on white ≥ 5.5:1) — both AA-compliant. Purple accent (`#6C3CE0`) on white ≥ 7:1.
- Do NOT rely on color alone to encode chart meaning — include legend text + tooltip numeric values for every chart.
