    **1. Create `apps/web/lib/socket-client.ts`:**
    ```typescript
    import { io, Socket } from 'socket.io-client';

    const API_URL = process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8080';

    export function createBookingSocket(): Socket {
      return io(`${API_URL}/booking`, {
        transports: ['websocket', 'polling'],
        withCredentials: true,
        autoConnect: false,
        reconnection: true,
        reconnectionAttempts: Infinity,
        reconnectionDelay: 1000,
        reconnectionDelayMax: 5000,
      });
    }
    ```
    - Factory function, NOT a singleton. Each booking page mount creates a new socket.
    - `autoConnect: false` — hook controls connection lifecycle.
    - `withCredentials: true` — sends cookies for auth (matches backend CORS config).
    - `reconnection: true` with exponential backoff — Socket.IO default behavior.

    **2. Create `apps/web/hooks/use-socket.ts`:**
    ```typescript
    export function useBookingSocket(showtimeId: string | null): void
    ```
    Implementation:
    - `useRef<Socket | null>` to hold socket instance.
    - `useEffect` dependent on `showtimeId`:
      - If showtimeId is null, do nothing (no socket needed until showtime selected).
      - Create socket via `createBookingSocket()`.
      - `socket.connect()`.
      - On `'connect'`: call `useBookingStore.getState().setConnected(true)`. Emit `'join-showtime'` with showtimeId. Dismiss any disconnect toast.
      - On `'disconnect'`: call `useBookingStore.getState().setConnected(false)`. Show persistent toast via `toast.loading('실시간 연결이 끊어졌습니다. 재연결 중...', { id: 'ws-status' })` (sonner with fixed ID to prevent duplicates).
      - On `'seat-update'` event (data: `SeatUpdateEvent`):
        1. Get `queryClient` via `useQueryClient()`.
        2. Update React Query cache directly: `queryClient.setQueryData(['seat-status', showtimeId], (old: SeatStatusResponse | undefined) => { if (!old) return old; return { ...old, seats: { ...old.seats, [data.seatId]: data.status } }; })`.
        3. If `data.status === 'locked'` and the seatId is in `useBookingStore.getState().selectedSeats`: this is a race condition. Remove from store via `removeSeat(data.seatId)` and show toast "이미 다른 사용자가 선택한 좌석입니다" (per D-13).
      - On reconnect (`'connect'` after previous disconnect): toast.success('실시간 연결이 복구되었습니다', { id: 'ws-status', duration: 3000 }). Call `queryClient.invalidateQueries({ queryKey: ['seat-status', showtimeId] })` to refetch full state and catch missed events (per UI-SPEC WebSocket reconnect behavior).
      - Cleanup: `socket.emit('leave-showtime', showtimeId)`, `socket.disconnect()`, set ref to null.
    - **Important:** Access Zustand store via `getState()` (not hook) inside event handlers to avoid stale closures.

    **3. Create `apps/web/hooks/use-countdown.ts`:**
    ```typescript
    export function useCountdown(expiresAt: number | null, onExpire: () => void): { minutes: number; seconds: number; isWarning: boolean; isActive: boolean }
    ```
    Implementation:
    - If `expiresAt` is null, return `{ minutes: 0, seconds: 0, isWarning: false, isActive: false }`.
    - `useState` for `remaining` (in seconds).
    - `useEffect` with `setInterval(1000)`:
      1. Calculate `remaining = Math.max(0, Math.floor((expiresAt - Date.now()) / 1000))`.
      2. If remaining <= 0, call `onExpire()` and clear interval.
      3. Set state with new remaining.
    - Derive: `minutes = Math.floor(remaining / 60)`, `seconds = remaining % 60`.
    - `isWarning = remaining > 0 && remaining <= 180` (3 minutes = 180 seconds, per D-10).
    - `isActive = expiresAt !== null && remaining > 0`.
    - Clean up interval on unmount.
    - Use `useRef` for the onExpire callback to avoid stale closure issues (update ref on each render, call ref.current in interval).
