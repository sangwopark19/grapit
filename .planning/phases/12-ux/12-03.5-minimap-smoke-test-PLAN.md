**Check 1 PASS 기준:** 미니맵 박스 안에 축소 SVG가 정상 렌더링됨.
**Check 1 FAIL 시:** 박스는 있지만 내부가 비어있거나 색상이 깨지거나 스타일이 이상함. → react-zoom-pan-pinch MiniMap이 자식 render를 지원하지 않거나 클래스/스타일 전파가 기대와 다름. D-14 원안 fallback 필요.

**Check 2 — viewport rect 동기화:**
1. 메인 좌석맵에서 마우스 휠 up 또는 `+` 버튼으로 zoom in
2. 미니맵 안에 **빨간/Purple stroke의 viewport rect** (현재 보이는 영역을 표시하는 사각형)가 나타나는지 확인
3. 좌석맵을 드래그(pan)하면 viewport rect가 **실시간으로** 위치를 따라 움직이는지 확인
4. zoom out 하면 viewport rect가 커지거나 미니맵 전체 크기로 확장되는지 확인
5. flicker/lag이 없는지 (60fps 근접)

**Check 2 PASS 기준:**
- viewport rect가 정상 렌더링됨 (Brand Purple #6C3CE0 stroke)
- pan/zoom 시 실시간 동기화

**Check 2 FAIL 시:** viewport rect가 보이지 않거나, 동기화가 안 되거나, 심한 lag 발생. → D-14 원안 fallback 필요 — `TransformWrapper` `onTransformed` 콜백으로 positionX/Y/scale 구독 + 수동 `<rect>` 렌더.

──────

**Check 3 — 모바일 breakpoint 숨김:**
1. Chrome DevTools → Toggle device toolbar (Ctrl+Shift+M) → iPhone 12 등 모바일 디바이스 프리셋 선택
2. 좌석맵 페이지 새로고침
3. **미니맵 박스가 미마운트** 확인 (보이지 않아야 함)
4. Tailwind md breakpoint (`< 768px`) 기준 useIsMobile() === true → MiniMap 조건부 렌더링 X

**Check 3 PASS 기준:** 모바일 viewport에서 미니맵 박스가 DOM에 없음 (Elements 패널에서 `data-testid="minimap"` 또는 MiniMap 관련 요소 부재).
**Check 3 FAIL 시:** 미니맵이 모바일에서도 표시됨 → Plan 12-03 Task 1의 `{!isMobile && ...}` 조건부 렌더링 또는 useIsMobile 값이 잘못 작동. Plan 12-02/12-03 revision.

**FAIL 시 fallback 경로 (D-14 원안):**
Plan 12-03 Task 1의 MiniMap 사용을 제거하고 다음과 같이 수정:
1. `import { MiniMap }` 제거
2. TransformWrapper 직전에 `const [transformState, setTransformState] = useState({ positionX: 0, positionY: 0, scale: 1 });` 추가
3. `<TransformWrapper onTransformed={(ref, state) => setTransformState({ positionX: state.positionX, positionY: state.positionY, scale: state.scale })} ...>` 구성
4. `{!isMobile && (<div className="absolute top-3 left-3 z-40 ..."><svg dangerouslySetInnerHTML={{ __html: processedSvg }} style={{ width: 120, height: 'auto' }} />{/* 수동 viewport rect */}</div>)}`
5. viewport rect는 `transformState` 기반으로 `<rect>`를 SVG 위에 오버레이 (상세 좌표 계산은 D-14 원안 RESEARCH §Pattern 2 참고)

FAIL 구체 케이스는 Plan 12-03 Task 1 revision에서 결정.
