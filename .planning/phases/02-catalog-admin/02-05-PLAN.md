Fix 3 issues diagnosed in UAT Gap 2:

**1. Add keepPreviousData to useSearch hook (use-search.ts):**
Import `keepPreviousData` from `@tanstack/react-query` and add it as `placeholderData` option. This keeps the previous query result visible while the new query fetches, preventing data from going undefined on chip toggle.

Before:
```typescript
import { useQuery } from '@tanstack/react-query';
```
After:
```typescript
import { useQuery, keepPreviousData } from '@tanstack/react-query';
```

Add `placeholderData: keepPreviousData,` to the useQuery options object (after the `enabled` line).

Before:
```typescript
  return useQuery({
    queryKey: ['search', q, genre, ended, page],
    queryFn: () => { ... },
    enabled: q.length > 0,
  });
```
After:
```typescript
  return useQuery({
    queryKey: ['search', q, genre, ended, page],
    queryFn: () => { ... },
    enabled: q.length > 0,
    placeholderData: keepPreviousData,
  });
```

**2. Remove duplicate empty-state from search/page.tsx (lines 95-127):**
The current code has a ternary with 3 branches: error / empty / grid. The empty branch (lines 109-118) duplicates what PerformanceGrid already handles internally. This causes layout shift because the inline empty-state has different padding than PerformanceGrid's.

Replace the entire triple-ternary block (lines 95-127) with the genre page pattern — only error vs PerformanceGrid:

Before:
```tsx
<div className="mt-6">

  ) : data && data.data.length === 0 && !isLoading ? (
    <div className="flex flex-col items-center py-16">...</div>
  ) : (
    <PerformanceGrid ... />
  )}
</div>
```

After:
```tsx
<div className="mt-6">
  {isError ? (
    <div className="flex flex-col items-center py-16">

      </p>
      <button
```

**3. Always render total count container (search/page.tsx lines 64-66):**
The total count currently conditionally renders only when `data` exists. When keepPreviousData is active, `data` will remain defined during refetch, so the condition is less problematic. But for robustness, always render the container with a min-height placeholder.

Before:
```tsx
{data && (
  <p className="mt-1 text-sm text-gray-600">총 {data.total}건</p>
)}
```

After:
```tsx
<p className="mt-1 h-5 text-sm text-gray-600">
  {data ? `총 ${data.total}건` : '\u00A0'}
</p>
```

The `h-5` (20px) matches text-sm line height, reserving space even when data is undefined. `\u00A0` (non-breaking space) keeps the element from collapsing.
