**Case A: HTTP 200 + accessToken 반환 + user.role === "admin"**

**Case B: HTTP 401 (이메일 또는 비밀번호 불일치)**
- 프로덕션 DB에 admin@grapit.test 계정이 없거나 password_hash가 다름
- Cloud SQL Auth Proxy를 통해 프로덕션 DB에 접속하여 확인:
  1. `gcloud sql connect` 또는 Cloud SQL Auth Proxy 시작
  2. `SELECT id, email, role, password_hash IS NOT NULL as has_password FROM users WHERE email = 'admin@grapit.test';` 로 확인
  3-a. 계정이 없으면: seed.mjs의 admin INSERT 부분만 프로덕션 DB에 직접 실행
  3-b. 계정이 있지만 role이 'user'이면: `UPDATE users SET role = 'admin' WHERE email = 'admin@grapit.test';`
  3-c. 계정이 있지만 password_hash가 없으면: argon2id로 해싱한 비밀번호를 UPDATE

프로덕션 DB 접속 방법 (Cloud SQL Auth Proxy):
```bash
# 방법 1: gcloud sql connect (가장 간단)
gcloud sql connect [INSTANCE_NAME] --user=postgres --database=grapit

# 방법 2: Cloud SQL Auth Proxy
cloud-sql-proxy [PROJECT_ID:asia-northeast3:INSTANCE_NAME] --port=5433 &
psql "postgresql://postgres:PASSWORD@127.0.0.1:5433/grapit"
```

인스턴스 이름 확인:
```bash
gcloud sql instances list
```

-- argon2 해싱은 Node.js에서 해야 하므로, seed.mjs를 프로덕션 DB에 실행
-- DATABASE_URL을 프로덕션으로 설정하고 seed의 admin 부분만 실행
-- 또는 별도 스크립트 작성

seed.mjs를 프로덕션 DB에 실행하면 모든 데이터가 초기화되므로 주의. admin INSERT만 별도 실행하는 것이 안전:
```bash
# 프로덕션 DB URL로 admin만 INSERT하는 1회성 스크립트 실행
DATABASE_URL="프로덕션URL" node -e "
import pg from 'pg';
import argon2 from 'argon2';
const pool = new pg.Pool({ connectionString: process.env.DATABASE_URL });
const client = await pool.connect();
try {
  const hash = await argon2.hash('TestAdmin2026!', { type: argon2.argon2id, memoryCost: 19456, timeCost: 2, parallelism: 1 });
  await client.query(\`
    INSERT INTO users (id, email, password_hash, name, phone, gender, country, birth_date, role, is_phone_verified, is_email_verified)
    VALUES (gen_random_uuid(), 'admin@grapit.test', \\\$1, '관리자', '010-0000-0000', 'unspecified', 'KR', '1990-01-01', 'admin', true, true)
    ON CONFLICT (email) DO UPDATE SET role = 'admin', password_hash = \\\$1
  \`, [hash]);
  console.log('Admin user upserted');
} finally { client.release(); await pool.end(); }
"
```
