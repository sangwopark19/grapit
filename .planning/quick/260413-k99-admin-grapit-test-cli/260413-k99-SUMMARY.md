# Quick Task 260413-k99: 프로덕션 어드민 접속 불가 수정 - Summary

**Completed:** 2026-04-13
**Status:** Complete (pending deploy)

## What was done

### Root Cause Analysis

프로덕션에서 admin@grapit.test 계정으로 `/admin` 페이지에 접속할 수 없던 원인:

1. **API 로그인은 정상** — `curl POST /api/v1/auth/login` → HTTP 200, role=admin 확인
2. **쿠키 도메인 불일치** — `refreshToken` 쿠키가 API 도메인(`grapit-api-d3c6wrfdbq-du.a.run.app`)에만 설정됨. Domain 속성 없이 `SameSite=None; Secure; HttpOnly`로 설정
3. **proxy.ts 서버사이드 리다이렉트** — `apps/web/proxy.ts`가 `/admin` 접근 시 `refreshToken` 쿠키 존재 여부를 체크하고, 없으면 `/auth`로 307 리다이렉트
4. **크로스 도메인 쿠키 불가** — 웹 도메인(`grapit-web-d3c6wrfdbq-du.a.run.app`)에서는 API 도메인의 쿠키를 볼 수 없으므로 항상 리다이렉트됨

### Fix Applied

- `apps/web/proxy.ts`: 서버사이드 쿠키 체크 제거, pass-through로 변경
- 클라이언트 사이드 `admin/layout.tsx`의 auth 체크가 이미 동일한 보호를 제공하므로 보안 영향 없음

### Files Changed

| File | Change |
|------|--------|
| `apps/web/proxy.ts` | refreshToken 쿠키 체크 제거, NextResponse.next() pass-through |
