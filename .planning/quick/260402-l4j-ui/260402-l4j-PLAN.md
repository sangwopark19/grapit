<objective>
Tailwind CSS v4의 `--spacing-*` 토큰 충돌로 인해 모달 최대 너비가 24px(max-w-lg), 4px(max-w-xs), 8px(max-w-sm)으로 렌더링되는 버그를 수정한다.

Purpose: Tailwind v4에서 `--spacing-lg`, `--spacing-sm`, `--spacing-xs` 커스텀 토큰이 `max-w-lg/sm/xs` 유틸리티를 덮어쓰는 현상 해결. named max-width 유틸리티를 arbitrary value 표기(`[32rem]`)로 교체하면 spacing 토큰의 영향을 받지 않는다.
Output: 세 컴포넌트 파일 수정 완료, 모달 UI 정상 표시
</objective>

    AlertDialogContent 컴포넌트 className(61번 줄)에서 두 max-w 클래스를 교체한다.

    - `data-[size=sm]:max-w-xs` → `data-[size=sm]:max-w-[20rem]`
    - `data-[size=default]:sm:max-w-lg` → `data-[size=default]:sm:max-w-[32rem]`

    Tailwind v4의 spacing 토큰(`--spacing-xs: 4px`, `--spacing-lg: 24px`)이 named max-width 유틸리티를 덮어써서 모달이 찌그러지는 버그 수정. arbitrary value는 spacing 토큰의 영향을 받지 않는다.

    다른 className은 변경하지 않는다.
  </action>

</task>
