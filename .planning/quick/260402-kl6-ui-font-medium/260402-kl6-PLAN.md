1. Get the seat element's bounding box via `el.getBBox()` (SVGGraphicsElement method) — this gives x, y, width, height of the seat rect/circle.
2. Create a `<text>` element (namespace `http://www.w3.org/2000/svg`) with Unicode checkmark character `\u2713`.
3. Set attributes: `x` = center of bbox, `y` = center of bbox, `text-anchor="middle"`, `dominant-baseline="central"`, `fill="white"`, `font-size="12"`, `font-weight="bold"`, `pointer-events="none"`.
4. Insert the `<text>` element as a sibling AFTER the seat element (use `el.parentNode.insertBefore(checkEl, el.nextSibling)`).

Important: The DOMParser context is off-screen so `getBBox()` may not work. Instead, read x/y/width/height attributes directly if the seat is a `<rect>`, or cx/cy/r if `<circle>`. Fallback approach:
- For `<rect>`: x + width/2, y + height/2
- For `<circle>`: cx, cy
- For other shapes: try `el.getAttribute('x')` and `el.getAttribute('y')` with width/height, or skip checkmark if geometry cannot be determined.

**Fix 4 — Timer expired modal size** in `timer-expired-modal.tsx:20`:

Remove `size="sm"` from `<AlertDialogContent size="sm">` so it becomes `<AlertDialogContent>`. The `size="sm"` prop causes `AlertDialogFooter` to apply `grid-cols-2`, but the modal has only one button ("처음으로"), making it appear in the left half of a 2-column grid. Since this modal does NOT use AlertDialogFooter (it uses a custom flex layout), the actual visual issue is that `data-[size=sm]:max-w-xs` makes the modal too narrow. Remove `size="sm"` to use the default `sm:max-w-lg` width.
