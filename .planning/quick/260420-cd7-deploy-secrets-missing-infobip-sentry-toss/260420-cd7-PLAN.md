# Quick Task 260420-cd7 — PLAN

## Problem
Phase 10.1 머지(`9ffadce`) 직후 Deploy workflow(run 24646423766)가 실패. 실패 원인: `grapit-api-00018-j2z` 리비전이 기동 직후 `exit(1)`. Cloud Run 로그 기준 원인은 `SmsService` constructor 의 production 가드:

```
ERROR [ExceptionHandler] Error: [sms] INFOBIP_API_KEY, INFOBIP_BASE_URL, INFOBIP_SENDER required in production. Silent dev mock disabled.
    at new SmsService (/app/dist/modules/sms/sms.service.js:150:19)
Container called exit(1).
Default STARTUP TCP probe failed 1 time consecutively for container "grapit-api-1" on port 8080. The instance was not started.
```

Phase 10.1 에서 SMS provider 를 Twilio → Infobip 으로 전환하며 production 에서 3개 env (`INFOBIP_API_KEY`, `INFOBIP_BASE_URL`, `INFOBIP_SENDER`) 를 필수 검증으로 바꿨으나 배포 파이프라인(GCP Secret Manager + deploy.yml) 에 반영되지 않음. 동시에 `SENTRY_DSN`, `TOSS_SECRET_KEY`, `NEXT_PUBLIC_SENTRY_DSN`, `NEXT_PUBLIC_TOSS_CLIENT_KEY` 도 프로덕션에 전혀 주입되고 있지 않음을 확인.

## Strategy
- **api 런타임 시크릿 (5개)** → GCP Secret Manager 에 신규 저장 후 `deploy.yml` `deploy-api.secrets:` 블록에 참조 추가.
- **web 빌드타임 NEXT_PUBLIC_* (2개)** → GitHub Secrets 에 신규 저장 후 `deploy.yml` `deploy-web` Docker build step 에 `--build-arg` 추가. `NEXT_PUBLIC_*` 는 Next.js 번들에 빌드 시점 인라인이므로 Cloud Run runtime env 로 주입해도 브라우저에 도달하지 않음.
- `apps/web/Dockerfile` 의 builder stage 에 `ARG NEXT_PUBLIC_SENTRY_DSN`, `ARG NEXT_PUBLIC_TOSS_CLIENT_KEY` 추가.
- IAM: `grapit-cloudrun@grapit-491806.iam.gserviceaccount.com` 가 프로젝트 레벨에서 `roles/secretmanager.secretAccessor` 를 이미 보유 → 신규 시크릿 개별 바인딩 불필요 (확인 완료).

## Steps
1. Secret Manager 에 5개 시크릿 생성 + 값 업로드
   - `infobip-api-key`, `infobip-base-url`, `infobip-sender`, `sentry-dsn`, `toss-secret-key`
2. GitHub Secrets 에 2개 업로드
   - `NEXT_PUBLIC_SENTRY_DSN`, `NEXT_PUBLIC_TOSS_CLIENT_KEY`

6. Cloud Run 서비스 `Ready=True` 검증
