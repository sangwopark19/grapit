<objective>
Fix three Redis seat-locking bugs: (1) stale user-seats entries blocking new locks after TTL expiry,
(2) timer reset leaving orphaned Redis locks, (3) non-atomic seat count check allowing >4 locks via race condition.

Purpose: Without these fixes, users get permanently blocked from selecting seats after TTL expiry (Bug 1),
ghost-locked seats appear unavailable to all users after timer reset (Bug 2), and concurrent requests
can bypass the 4-seat limit (Bug 3).

Output: Corrected booking.service.ts with Lua-atomic lockSeat, new unlockAllSeats endpoint, frontend integration.
</objective>

    **Step 3 — Implement Lua-atomic lockSeat** (booking.service.ts):
    Replace the current multi-step lockSeat with a single `this.redis.eval()` call.
    The Lua script does:
    ```
    -- KEYS[1] = user-seats:{showtimeId}:{userId}
    -- KEYS[2] = seat:{showtimeId}:{seatId}
    -- KEYS[3] = locked-seats:{showtimeId}
    -- ARGV[1] = userId
    -- ARGV[2] = LOCK_TTL (600)
    -- ARGV[3] = MAX_SEATS (4)
    -- ARGV[4] = seatId
    -- ARGV[5] = key prefix "seat:{showtimeId}:"

    -- 1. Get all members of user-seats
    local members = redis.call('SMEMBERS', KEYS[1])
    -- 2. Check each member's lock key still exists, remove stale
    local alive = 0
    for i, sid in ipairs(members) do
      if redis.call('EXISTS', ARGV[5] .. sid) == 1 then
        alive = alive + 1
      else
        redis.call('SREM', KEYS[1], sid)
        redis.call('SREM', KEYS[3], sid)
      end
    end
    -- 3. Check count
    if alive >= tonumber(ARGV[3]) then
      return {0, 'MAX_SEATS'}
    end
    -- 4. SET NX
    local ok = redis.call('SET', KEYS[2], ARGV[1], 'NX', 'EX', tonumber(ARGV[2]))
    if not ok then
      return {0, 'CONFLICT'}
    end
    -- 5. SADD + EXPIRE
    redis.call('SADD', KEYS[1], ARGV[4])
    redis.call('EXPIRE', KEYS[1], tonumber(ARGV[2]))
    redis.call('SADD', KEYS[3], ARGV[4])
    return {1, KEYS[2], ARGV[4]}
    ```
    Parse the result array: `[1, lockKey, seatId]` = success, `[0, 'MAX_SEATS']` = throw ConflictException with max seats message, `[0, 'CONFLICT']` = throw ConflictException with seat taken message.
    After successful eval, call `this.gateway.broadcastSeatUpdate(...)` (broadcast stays in Node, not Lua).
