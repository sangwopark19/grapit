**Root cause:** `useFieldArray`의 `fields`는 append/remove 시점의 snapshot이다. `setValue('castings.${index}.photoUrl', publicUrl)` 호출 후에도 `field.photoUrl`은 여전히 null — 이미지 미리보기가 표시되지 않음.

**Fix 1 — casting-manager.tsx:**

1. Props 인터페이스에 `control: Control<CreatePerformanceFormInput>` 추가 (react-hook-form의 `Control` 타입 import)
2. 각 캐스팅 카드를 별도 컴포넌트 `CastingCard`로 추출:
   - props: `index`, `field`, `control`, `register`, `remove`, `onPhotoUpload`
   - 내부에서 `const watchedPhotoUrl = useWatch({ control, name: \`castings.${index}.photoUrl\` })` 호출
   - 이미지 표시 조건을 `field.photoUrl`에서 `watchedPhotoUrl`로 변경
3. 포스터 업로드와 동일한 blob URL 즉시 미리보기 패턴 적용:
   - `handlePhotoUpload`에서 `const blobUrl = URL.createObjectURL(file)` 생성
   - blob URL을 로컬 state로 관리 (각 CastingCard 내부에 `const [preview, setPreview] = useState<string | null>(null)`)
   - 업로드 시작 시 `setPreview(blobUrl)`, 실패 시 `setPreview(null)` + `URL.revokeObjectURL(blobUrl)`
   - 이미지 표시: `preview ?? watchedPhotoUrl` 사용 (blob 우선, 없으면 서버 URL)
4. 사진이 이미 있을 때도 클릭으로 교체 가능하도록: 이미지 위에 클릭 핸들러 추가하여 file input 트리거

**Root cause:** `admin.service.ts` line 42에서 `onConflictDoUpdate({ target: venues.name })`를 사용하지만, `venues` 테이블의 `name` 컬럼에 UNIQUE constraint가 없어 PostgreSQL이 `ON CONFLICT (name)` 실행 시 에러 발생.

**Fix — venues.ts:**

1. `name` 컬럼에 `.unique()` 추가:
   ```typescript
   name: varchar('name', { length: 255 }).notNull().unique(),
   ```

**Migration 생성:**

1. 루트에서 실행: `DOTENV_CONFIG_PATH=../../.env pnpm --filter @grapit/api exec drizzle-kit generate`
2. 생성된 migration SQL 파일 확인 — `ALTER TABLE "venues" ADD CONSTRAINT ... UNIQUE ("name")` 또는 `CREATE UNIQUE INDEX` 포함 확인
3. migration 적용: `DOTENV_CONFIG_PATH=../../.env pnpm --filter @grapit/api exec drizzle-kit migrate`

**주의:** `admin.service.ts`는 수정 불필요 — `onConflictDoUpdate({ target: venues.name })` 코드 자체는 올바르며, DB에 UNIQUE constraint만 추가하면 정상 동작.
