   async saveLocalFile(key: string, buffer: Buffer): Promise<string> {
     const uploadDir = path.join(process.cwd(), 'uploads');
     const filePath = path.join(uploadDir, key);
     await fs.mkdir(path.dirname(filePath), { recursive: true });
     await fs.writeFile(filePath, buffer);

   async getLocalFile(key: string): Promise<{ buffer: Buffer; contentType: string } | null> {
     const filePath = path.join(process.cwd(), 'uploads', key);
     try {
       const buffer = await fs.readFile(filePath);
       const ext = path.extname(key).slice(1);
       const mimeMap: Record<string, string> = { jpg: 'image/jpeg', jpeg: 'image/jpeg', png: 'image/png', gif: 'image/gif', svg: 'image/svg+xml', webp: 'image/webp' };

**admin-performance.controller.ts 수정:**

1. 로컬 파일 업로드 엔드포인트 추가. `@Put('upload/local/:folder/:filename')`으로 PUT 요청을 받아 파일을 저장한다. 프론트엔드의 presigned URL PUT 플로우와 동일한 방식(raw body PUT)을 사용하므로 multer가 아닌 raw body를 사용한다:
   ```typescript
   @Put('upload/local/:folder/:filename')
   async uploadLocal(
     @Param('folder') folder: string,
     @Param('filename') filename: string,
     @Req() req: RawBodyRequest<Request>,
   ) {
     const key = `${folder}/${filename}`;
     const chunks: Buffer[] = [];
     // NestJS에서 raw body를 읽기 위해 request stream 사용
     // 또는 express raw body middleware를 활용
   }
   ```
   실제로는 NestJS의 `@nestjs/platform-express`에서 `RawBodyRequest`를 사용하거나, 더 간단하게 `@UseInterceptors(FileInterceptor('file'))` + `@UploadedFile()`을 사용하되, 프론트엔드가 presigned URL과 동일하게 PUT raw body를 보내므로 raw body 파싱이 필요하다.

   **가장 간단한 접근**: main.ts에서 `app.useBodyParser('raw', { type: 'image/*', limit: '10mb' })` 를 추가하지 말고, 컨트롤러에서 request stream을 직접 읽는 방식을 사용:
   ```typescript
   @Put('upload/local/:folder/:filename')
   async uploadLocal(
     @Param('folder') folder: string,
     @Param('filename') filename: string,
     @Req() req: Request,
   ): Promise<{ success: true }> {
     const key = `${folder}/${filename}`;
     const buffers: Uint8Array[] = [];
     for await (const chunk of req) {
       buffers.push(chunk);
     }
     const buffer = Buffer.concat(buffers);
     await this.uploadService.saveLocalFile(key, buffer);
     return { success: true };
   }
   ```

2. 로컬 파일 서빙 엔드포인트 추가. `@Get('upload/local/:folder/:filename')`으로 저장된 파일을 반환:
   ```typescript
   @Get('upload/local/:folder/:filename')
   async serveLocal(
     @Param('folder') folder: string,
     @Param('filename') filename: string,
     @Res() res: Response,
   ) {
     const key = `${folder}/${filename}`;
     const file = await this.uploadService.getLocalFile(key);
     if (!file) {
       res.status(404).json({ error: 'File not found' });
       return;
     }
     res.setHeader('Content-Type', file.contentType);
     res.send(file.buffer);
   }
   ```
   `@Res()` 사용 시 `import { Response } from 'express'`.

3. 로컬 파일 서빙 GET 엔드포인트는 `@Roles('admin')` guard를 제거해야 한다. 이미지 URL이 `<img src=...>`로 사용되므로 인증 없이 접근 가능해야 한다. 방법: 별도 컨트롤러를 만들지 말고, 클래스 레벨 `@Roles('admin')`을 개별 메서드로 이동하거나, serveLocal 메서드에 `@Public()` 데코레이터를 적용한다. 기존 코드를 확인해서 `@Public()` 데코레이터가 있는지 확인하고, 없으면 serveLocal에서만 guard를 우회하는 방식을 사용한다.

   **실용적 접근**: `upload/local` GET 라우트를 AdminPerformanceController 외부, 별도 간단한 컨트롤러 `LocalUploadController`로 만든다. 이 컨트롤러에는 `@Roles`를 달지 않는다. AdminModule에 등록한다.

   그러나 더 간단하게: AdminPerformanceController의 클래스 레벨 `@UseGuards(RolesGuard)` `@Roles('admin')`이 있으므로, serveLocal GET 엔드포인트만 별도 파일 `local-upload.controller.ts`로 분리하는 게 깔끔하다. 하지만 파일 수를 최소화하기 위해, serveLocal에 guard를 명시적으로 비활성화하는 방법이 더 좋다.

   **최종 결정**: GET 엔드포인트를 AdminPerformanceController에 넣되, RolesGuard가 토큰이 없어도 통과하게 동작하는지 확인한다. 현실적으로 로컬 dev에서만 사용되므로, GET 엔드포인트도 admin guard 안에 두고 프론트엔드에서 img 표시 시에는 `posterPreview`를 blob URL이나 data URL로 처리한다. 아, 그런데 performance-form.tsx를 보면 `setPosterPreview(publicUrl)`로 publicUrl을 직접 img src에 사용한다. 로컬 모드에서는 publicUrl을 NestJS 서버의 URL로 설정하므로 인증 쿠키가 같은 origin이면 동작할 수 있다.

   **최종 접근**: GET 엔드포인트도 AdminPerformanceController에 같이 두되, 로컬 dev에서 img 로드 시 쿠키가 함께 전송되므로 RolesGuard를 통과할 수 있다. 만약 통과하지 못하면 프론트엔드에서 posterPreview를 upload 시 `URL.createObjectURL(file)`로 설정하는 게 더 깔끔하다. 이 방식은 Task 2에서 처리.

**performance-form.tsx 수정:**

1. `handlePosterUpload` 수정: presigned 응답의 `mode` 값에 따라 분기 처리.
   - `mode === 'local'`: uploadUrl로 PUT 요청 시 `credentials: 'include'`를 추가한다 (admin guard 통과를 위해 쿠키 전송). 기존 코드의 `fetch(uploadUrl, { method: 'PUT', body: file, headers: { 'Content-Type': file.type } })`에 credentials 추가.
   - `mode === 'r2'` (또는 mode가 없는 경우 하위 호환): 기존 로직 그대로 유지 (R2 presigned URL은 credentials 불필요).
2. 포스터 미리보기를 즉시 표시하기 위해 `setPosterPreview`를 `URL.createObjectURL(file)`로 먼저 설정하고, 업로드 성공 후에 `publicUrl`로 교체한다. 이렇게 하면 로컬 모드에서 img src가 인증 필요한 URL이어도 미리보기는 blob URL로 잘 보인다:
   ```typescript
   const handlePosterUpload = useCallback(
     async (file: File) => {
       if (file.size > 5 * 1024 * 1024) {
         toast.error('포스터 이미지는 5MB 이하여야 합니다.');
         return;
       }
       // 즉시 미리보기 표시
       const blobUrl = URL.createObjectURL(file);
       setPosterPreview(blobUrl);

       const ext = file.name.split('.').pop() ?? 'jpg';
       try {
         const { uploadUrl, publicUrl, mode } =
           await presignedUpload.mutateAsync({
             folder: 'posters',
             contentType: file.type,
             extension: ext,
           });
         await fetch(uploadUrl, {
           method: 'PUT',
           body: file,
           headers: { 'Content-Type': file.type },
           ...(mode === 'local' ? { credentials: 'include' as const } : {}),
         });
         form.setValue('posterUrl', publicUrl, { shouldDirty: true });
         // blob URL 해제 (메모리 누수 방지), publicUrl로 교체하지 않고 blobUrl 유지
         // 폼 제출 시 publicUrl이 서버에 전달되므로 미리보기는 blobUrl로 충분
         toast.success('포스터가 업로드되었습니다.');
       } catch {
         URL.revokeObjectURL(blobUrl);
         setPosterPreview(posterPreview ?? '');  // 이전 상태 복원
         toast.error('포스터 업로드에 실패했습니다.');
       }
     },
     [form, presignedUpload, posterPreview],
   );
   ```
