## 1. 서비스 개요

| 항목 | 내용 |

| 미션 | “좋아하는 순간을 Grab it” — 팬미팅·팬사인회·팝업 입장까지 가장 설레는 예약 경험 제공 |

### 1.1 경쟁자 대비 Grab it 포지션

| 플랫폼 | 성격 | 핵심 한계 |

| 구글폼+통장이체 | 무료 수동 파이프라인 | 결제 검증 없음 / 추첨 불신 / 정산·세금계산서 불가 |
|  |  |  |

**Grab it이 최초로 제공하는 것**
- 추첨 자동화 (공정성 증명 로그 영구 보관)
- 선착순 대기열 (1계정 3기기 정책)
- 예약자 명단 CSV 등급별 분리 다운로드
- 온라인 팝업 입장권 (링크+비밀번호+IP 비주얼 테마)
- 소규모(30~500명) 셀프서브 이벤트 개설 (V3)

## 3. 화면 공통 기반

### 3.1 GNB — 4탭 구조

**레이아웃: 3존(Zone A/B/C)**

- **Zone A**: GNB 탭 — **홈 / 티켓팅 / 팝업 / 오픈예정**. 항상 고정. 어느 화면에서든 접근 가능.
- **Zone B**: 1차 필터 칩 바 — 탭별 칩 구성만 다르고 가로 스크롤 칩 컴포넌트로 통일. Sticky. 홈·오픈예정 탭에는 없음.
- **Zone C**: 정렬·이벤트 수 한 줄 — **총 이벤트 수(좌) + 정렬 드롭다운 + 마감 이벤트 포함 토글(우)**. 티켓팅·팝업 탭에서 표시. 팝업 탭 오프라인 선택 시 지역 필터 추가.
- **모바일**: Zone B = 가로 스크롤 칩. Zone C = 한 줄 (PC와 동일 구조, 하단 시트 없음).

| 탭 | Zone B (1차 필터 칩) | Zone C (정렬·이벤트 수) | 히어로 슬라이더 |
| --- | --- | --- | --- |
| 홈 | 없음 (섹션 구조) | 없음 | ✅ |
| 티켓팅 | 전체 / 요즘 HOT / IP분류 / 예약방식(선착순·추첨) / 마감임박 / 지역 | 총 이벤트 수(좌) + 정렬(오픈순·인기순·등록순) + 마감 포함 토글 기본 OFF(우) | ✅ |
| 팝업 | 전체 / 요즘 HOT / IP분류 / 온라인 팝업 / 오프라인 팝업 / 마감임박 | 총 이벤트 수(좌) + 정렬(오픈순·인기순·등록순) + 마감 포함 토글 기본 OFF(우). 오프라인 선택 시 지역 필터 추가. | ✅ |

> 

```
결제 완료 (또는 무료 확정)
    ↓
예매완료 ─── 취소·환불 가능 (취소가능일 이내)
    │
    ├── 수동 확정: 사용자가 "예약확정" 버튼 클릭 → 확인 모달 → 즉시 확정

    ↓
예매확정 ─── 취소·환불 불가. 포인트 적립.
```

### 3.5 회원 인증

**회원가입**
- 방식: 이메일+비밀번호 / 카카오·네이버·구글 소셜 로그인
- 이메일 가입: 이메일 입력 → 인증 메일 → 링크 클릭 → 닉네임·비밀번호 설정
- 소셜 가입: 소셜 인증 → 닉네임 설정 → 완료 (이메일 자동 수집)
- 14세 미만 가입 불가 (개인정보보호법)

**로그인**
- 이메일+비밀번호 / 카카오·네이버·구글 소셜
- JWT 액세스+리프레시 토큰 / 자동 갱신 / 30일 유지

**본인 인증 (PASS) → 논의 필요**

- 모든 이벤트 필수
- 1인 1계정 — 동일 휴대폰 번호로 중복 가입 불가
- 관람연령 제한 이벤트 예매 시 성인 인증 자동 요구

**비밀번호 재설정**
- “비밀번호 찾기” → 이메일(ID)·이름·핸드폰번호 입력 → 인증번호 발송 (이메일 or 핸드폰 선택) → 인증번호 입력 → 새 비밀번호 설정

**당첨**
- 진행 단계 바: 응모(✓) → 추첨(✓) → 당첨 확인(진행중) → 결제 → 완료
- 결제 기한 안내. 기한 만료 시 당첨 자동 취소.
- 당첨 정보: 이벤트·관람일·회차·당첨 등급·매수·결제 금액
- 버튼: 지금 결제하기 (→ S-15) / 당첨 취소(포기)

**낙첨**
- 낙첨 안내 + 총 응모 경쟁률 (등급별 모집 수·응모 수 실제 공개)
- 유사 이벤트 추천 카드 (같은 IP·출연자 기준). 카드 클릭 → S-06.
- 버튼: 다음 이벤트 알림 신청 → S-06 우측 패널

### 7.3 S-15 당첨 후 결제

> **진입**: S-14 “지금 결제하기” / 마이페이지 ① 예약 내역 “결제하기” (당첨·결제 대기 상태) / S-06 우측 패널 “지금 결제하기” — 3가지 모두 동일 화면으로 연결.
> 
- 상단 고정: 결제 마감 기한 안내 + “기한 초과 시 자동 취소” 경고
- 당첨 등급 확인 배너: 이벤트·관람일·회차·등급 요약
- 결제 수단·쿠폰·포인트 적용 → 결제하기 → S-10 예약 완료

| 무료 선착순·추첨 당첨 | 이벤트 시작 3시간 전까지 취소. 취소 시 대기자 이전 (운영팀 옵션). |
| 온라인 팝업 | 입장 기간 시작 전까지 취소. 취소 시 대기자 순차 이전 (운영팀 옵션). |

**공통 규칙**

- **예매확정(수동·자동) 후에는 취소·환불 불가.** 확정 전 모달로 충분히 고지.
- 주최자 귀책 취소 (유료): 전액 환불 + 구매가 10% 배상 (예매확정 상태 포함)
- 주최자 귀책 취소 (무료·팝업): 공지 발송으로 대체

### 10.2 오픈 알림 신청

> "특정 이벤트" 단위 알림 구독. 출연자·작가 단위 팔로우(10.3)와 구별된다.
> 

> 카드에 별도 알림 신청 버튼 없음. 모든 오픈 알림 신청은 **S-06 우측 패널**에서 이루어짐.
> 
- **주요 진입점**: S-06 우측 패널 — 추후공지·오픈예정 상태에서 알림 신청 버튼 노출
- 홈 ④ / S-05 / 검색·랭킹 → 카드 클릭 → S-06 우측 패널에서 신청

**신청 플로우**

- [비로그인] 알림 신청 버튼 클릭 → 회원가입 → 로그인 → 신청 완료
- [로그인] 버튼 클릭 즉시 완료. "알림 신청 →" → "알림 신청됨 ✓". 마이페이지 ④ 오픈 알림 신청에 자동 추가.

| 발송 시점 | 내용 | 수단 |
| --- | --- | --- |
| 예매 오픈 24시간 전 | 이벤트명·오픈 일시·예매 바로가기 링크 | 이메일 + SMS |
| 예매 오픈 1시간 전 | 이벤트명·오픈 일시·예매 바로가기 링크 | 이메일 + SMS |

> 팔로우 알림과 오픈 알림이 동일 이벤트·동일 시점에 겹치면 **1회만 발송** (중복 방지).
> 

### 10.3 팔로우 기능

> 팔로우 대상은 **출연자·작가** 단위와 **IP** 단위 두 가지. 팔로우하면 관련 이벤트 오픈 시 알림을 받는다.
> 

**출연자·작가 팔로우**

| 항목 | 내용 |
| --- | --- |
| 팔로우 대상 | 출연자·작가·캐릭터 (V1). 주최자 팔로우는 V2. |
| 진입점 | S-06 ② 캐스팅 정보 탭 → 출연자·작가·캐릭터 프로필 카드 하단 "팔로우" 버튼. 클릭 즉시 완료. 비로그인 시 로그인 유도 모달. |
| 팔로우 해제 | 마이페이지 → 관심 목록 → 팔로우 탭 / 또는 캐스팅 정보 탭 "팔로잉 ✓" 재클릭 → 확인 모달 → 해제 |

| 알림 발송 트리거 | 내용 | 수단 |
| --- | --- | --- |
| 팔로우한 출연자·작가·캐릭터가 포함된 새 이벤트 발행 | 이벤트명·오픈 일시·CTA 링크 | 이메일 + SMS |
| 팔로우한 이벤트 예매 오픈 24시간 전 | 오픈 임박 리마인드 | 이메일 + SMS |
| 팔로우한 이벤트 예매 오픈 1시간 전 | 오픈 임박 리마인드 | 이메일 + SMS |

**IP 팔로우**

| 항목 | 내용 |
| --- | --- |
| 팔로우 대상 | 출연자·작가·캐릭터 + IP 단위 팔로우 (V1) / 주최자 팔로우 (V2) |
| 진입점 | S-06 좌측 패널 IP 태그 옆 "팔로우" 버튼 (인라인). 클릭 즉시 완료. 비로그인 시 로그인 유도 모달. |

| 팔로우 해제 | 마이페이지 → 관심 목록 → 팔로우 탭 (IP 서브탭) / 또는 S-06 좌측 패널 "팔로잉 ✓" 재클릭 |

| 알림 발송 트리거 | 내용 | 수단 |
| --- | --- | --- |

| 팔로우한 IP 이벤트 예매 오픈 24시간 전 | 오픈 임박 리마인드 | 이메일 + SMS |
| 팔로우한 IP 이벤트 예매 오픈 1시간 전 | 오픈 임박 리마인드 | 이메일 + SMS |

> 출연자·작가 팔로우 + IP 팔로우 + 오픈 알림 신청이 동일 이벤트·동일 시점에 겹치면 **1회만 발송** (중복 방지).

- 쿠폰 + 포인트: 항상 동시 적용 가능
- 1회 결제에 쿠폰 **1장만** 사용 가능
- 카드사 할인 + 쿠폰: 카드사 할인은 PG사에서 처리, 플랫폼 쿠폰과 독립 적용

- 주최자 귀책 취소 시 (예매확정 포함): 적립된 포인트 유지.

### 11.4 어드민 ② 이벤트 관리 - 상세

- 액션: 수정 / 복사 / 일괄 상태 변경 / 발행 (임시저장 한정)

**회차 관리**

- 캘린더·리스트 뷰 전환 / 날짜·시간·정원·상태 설정 / 일괄 등록 (패턴 기반)

**출연자·작가·캐릭터 관리**

- 유형 선택: 출연자 / 작가 / 캐릭터 (IP 내 등장 캐릭터 단위)
- 프로필 등록: 이름 / 사진 / 유형별 부가 정보 (출연자·작가: 소속·SNS / 캐릭터: 원작 IP·캐릭터 설명) / SNS 링크
- 회차별 캐스팅 배정: 체크박스 방식으로 캐스팅-회차 매핑
- 추후 공개 여부 설정: 미공개 시 사용자 화면에서 마스킹 ("추후공지")
- 캐스팅 일정표 발행: 발행 즉시 S-06 ② 캐스팅 정보 탭에 반영
- 캐스팅 변경 시 예약자 알림 발송 여부 선택 (이메일·SMS)

**후기 관리**

- 이벤트별 후기 목록: 작성자·별점·내용·작성일 / 평균 별점 표시
- 신고 접수 후기: 별도 필터로 분류 → 숨김·삭제 처리
- 부적절 내용(스팸·비방) 수동 숨김·삭제 / 처리 사유 기록

**공지사항 관리**

- 이벤트별 공지 CRUD: 유형(긴급·공지·일반) / 게시 상태
- 긴급 공지: 이벤트 상세 최상단 고정 + 사용자 알림 발송 (이메일+SMS)

**Q&A 관리**

- 미답변·답변완료 필터 / 미답변 건수 배지
- 답변 작성·수정·게시 즉시 노출 / 질문 숨기기·삭제
- V1에서는 운영팀 전담. 기획사 파트너(RBAC)는 V3에서 본인 이벤트 Q&A만 접근.

**선예매 관리**

- 이벤트별 선예매 활성화 여부 / 선예매 기간·대상 등급·가격 설정

**팔로워·오픈 알림 신청 현황**

- 이벤트별: 팔로워 수 (해당 이벤트에 출연·참여하는 출연자·작가·캐릭터 팔로워 + 관련 IP 팔로워 합산) / 오픈 알림 신청 수 (이메일·SMS 구분)
- 발송 이력: 발송 일시·수신자 수·실패 건수 / 실패 재발송

### 11.5 어드민 ③ 예약자 관리 - 상세

- 예매확정 상태 취소 불가 (주최자 귀책 취소는 예외 처리)

**예약확정 일괄 처리**

- 이벤트별 예매완료 상태 전체 → 예매확정 일괄 처리
- 수동 일괄 실행 버튼 / 또는 시스템 설정(⑫)의 자동 확정 N일과 연동하여 자동 처리
- 확정 처리 시 포인트 자동 적립 트리거 / 처리 결과 로그 기록

**좌석 현황 뷰**

- 이벤트·회차별 SVG 좌석 배치도 뷰: 예매 완료·잔여·비활성화 색상 구분
- 특정 좌석 강제 비활성화·활성화 처리 가능 (운영 이슈 발생 시)
- 명단 CSV에 좌석 번호 자동 포함

**명단 CSV**

- 컬럼: 예약번호 / 이름 / 연락처 / 등급 / 회차 / 좌석번호(수동배정시) / 결제금액 / 상태 / 예매일시
- 이벤트별·회차별·등급별 분리 다운로드 (팬사인회 현장 운영용)
- 다운로드 이력 기록 (보안 감사용)

**예매자 통계**

- 요약 대시보드: 총 예약 수 / 총 매출 / 취소율 / 환불 금액 / 성별 비율 / 연령대 분포

### 11.6 어드민 ④ 추첨 관리 - 상세

**당첨자 관리**

- 당첨자 목록: 이름·연락처·등급·결제 상태·결제 기한 잔여시간
- 결제 기한 임박 알림 수동 재발송
- 미결제 기한 초과: 자동 취소 → 차순위 당첨 자동 처리
- 차순위 당첨자 수동 지정 가능 (차순위 미지정 시 해당 등급 미충원)
- 당첨 취소(포기) 처리: 사용자 요청 접수 → 어드민 처리 → 차순위 이전

**낙첨 처리**

- 낙첨 알림 자동 발송 / 낙첨자 목록 확인
- 추첨 결과 통계: 등급별 당첨률·낙첨률 요약

### 11.7 어드민 ⑤ 팝업 관리 - 상세

**온라인 팝업**

예약 현황

- 확정자·잔여 정원·대기자 수 실시간 표시
- 대기자 목록: 순번·사용자명·신청일시 / 수동 입장 확정 가능

링크·비밀번호 발송

- 최초 발송: 입장 확정 즉시 자동 발송 (이메일·우편)
- 재발송: 사용자 요청 접수 → 어드민 수동 재발송 / 사용자가 마이페이지에서 직접 재발송 요청 시 자동 처리
- 발송 이력: 발송 일시·수신 여부·재발송 횟수

접근 로그·보안

- 접근 IP·사용 횟수·최근 접근 시각 기록
- 어뷰징 탐지: 비밀번호 3회 오류 → 1시간 잠금 / 동일 IP 다중 접근 이상 감지
- 잠금 수동 해제 가능

콘텐츠 구성

- IP 비주얼 테마 (대표색·배경·로고) 설정·미리보기
- 쇼핑몰 URL / 입장 페이지 콘텐츠 설정
- 입장 기간 연장·조기 종료 / 기간 변경 시 입장자 알림 발송 여부 선택

**오프라인 팝업**

예약 현황

- 방문일·시간대별 예약 현황 / 잔여 인원 실시간 표시
- 예약자 명단: 이름·연락처·방문일·시간대·인원·상태 / 방문일·시간대별 분리 다운로드
- V1: 현장 운영자가 마이페이지 예약 상세 화면 대조로 입장 처리
- V2a부터: QR 체크인 기능 추가

예약 관리

- 예약 취소 처리 / 시간대 변경 (운영팀 직접 처리)
- 잔여 정원 수동 조정 가능

### 11.8 어드민 ⑥ 정산 관리 - MVP 범위

> MVP에서는 정산 정보 등록 및 이벤트별 매출 요약 조회까지. 정산서 자동 생성은 V2a, 세금계산서·홈택스 연동은 V2b.
> 

**정산 정보 등록**

- 계좌: 은행 / 계좌번호 / 예금주
- 사업자 정보: 유형(개인·법인) / 사업자등록번호 / 상호 / 대표자
- 세금계산서 수신 이메일
- 원천징수 여부 (개인 주최자: 지급명세서 + 원천세 3.3%)
- 정산 정보 미등록 시 이벤트 발행 차단

**이벤트별 매출 요약 (V1 조회 전용)**

- 이벤트별 총 결제 금액 / 취소·환불 금액 / 순 매출
- 등급별 판매 수·금액 분류
- 플랫폼 수수료 차감 예상액 (수수료율 확정 후 적용)
- 상세 정산서 자동 생성은 V2a에서 활성화

### 11.9 어드민 ⑦ 배너 관리 - 상세

> 플랫폼 전반에 노출되는 배너·슬라이더 소재를 위치별로 등록·관리한다. **콘텐츠 담당자** 역할에서 관리.
> 

**관리 위치별 구성**

| 위치 | 형태 | 설명 |
| --- | --- | --- |
| 홈 메인 배너 | 풀와이드 자동 슬라이드 | 이미지(16:9) / 링크 URL / 노출 기간·순서 / 활성화 여부 |
| 티켓팅 탭 히어로 슬라이더 | 와이드형 자동 슬라이드 | 이벤트 직접 연결 or 외부 URL / 노출 기간·순서 |
| 팝업 탭 히어로 슬라이더 | 와이드형 자동 슬라이드 | 이벤트 직접 연결 or 외부 URL / 노출 기간·순서 |
| S-05 오픈예정 히어로 슬라이더 | 3열 와이드 수동 슬라이드 | 이벤트 직접 연결 / 핀 고정 여부 / 오픈 임박순 자동 정렬 기본값 |

**배너 등록 공통 항목**

- 소재: 이미지 업로드 (위치별 권장 해상도 안내) / 이벤트 직접 연결 시 이벤트 선택으로 이미지·링크 자동 채움
- 노출 기간: 시작·종료 일시 설정 / 기간 만료 시 자동 비활성화
- 순서: 드래그 앤 드롭 정렬 / 핀 고정은 S-05 슬라이더에서만 지원
- 활성화 토글: 기간 내에서도 수동 ON/OFF 가능
- 미리보기: 저장 전 PC·모바일 미리보기 지원

### 11.10 어드민 ⑧ 프로모션 - 상세

**쿠폰 관리**

- 쿠폰 목록: 쿠폰명 / 할인 유형(% 할인·정액) / 적용 범위 / 최소 결제 금액 / 사용 기간 / 발급 수·사용 수 현황
- 쿠폰 발급: 공개 코드 입력형 / 대상자 직접 발송형 (이메일·SMS)
- 적용 범위 설정: 전체 이벤트 / 특정 IP 카테고리 / 특정 이벤트 한정
- 이벤트 할인과 중복 허용 여부 설정
- 쿠폰 비활성화·삭제 / 사용 이력 조회

**포인트 설정**

- 예약확정 적립률: 결제 금액의 N% (기본 1%, 이벤트별 개별 설정 가능)
- 후기 작성 적립: 텍스트만 / 이미지 포함 각각 고정 포인트 설정 (후기 기능은 V2a에서 도입)
- 회원 가입 웰컴 포인트 설정
- 포인트 소멸 기준: 마지막 활동일 기준 N일 (기본 365일)

### 11.11 어드민 ⑨ CS 관리 - 상세

**문의 목록**

- 검색: 문의자명·이메일·예약번호·키워드
- 필터: 상태(접수·처리중·완료·보류) / 카테고리 / 담당자 / 기간
- 테이블 컬럼: 문의번호 / 문의자 / 카테고리 / 제목 / 접수일시 / 상태 / 담당자

**문의 카테고리**

- 예매·결제 / 취소·환불 / 추첨·당첨 / 팝업 입장 / 계정·회원 / 이벤트 정보 / 기타

**문의 상세**

- 문의 내용 + 첨부파일 / 예약 컨텍스트 자동 연동 (해당 예약번호 기준)
- 답변 작성 (내부 메모 / 사용자 발송 구분) / 답변 완료 시 이메일·SMS 자동 발송
- 담당자 배정·이관 / 처리 이력 로그

**상태 관리**

- 접수 → 처리중 → 완료 / 보류 (추가 확인 필요 시)
- 미처리 문의 알림: 접수 후 N시간 미답변 시 담당자 이메일 알림

### 11.12 어드민 ⑩ 공지 관리 - 상세

**플랫폼 공지**

- 공지 CRUD: 제목·내용(리치텍스트) / 게시 기간 / 중요도(일반·중요·긴급)
- 긴급 공지: 플랫폼 전체 상단 배너 고정 + 로그인 사용자 이메일 발송 여부 선택
- 예약 발행 지원 (일시 지정 노출)

**이벤트별 공지**

- ① 이벤트 관리에서 이벤트별 공지 관리와 연동 (여기서는 전체 목록 조회만)

**FAQ 관리**

- 카테고리별 FAQ CRUD: 예매 방법 / 취소·환불 / 추첨 / 팝업 입장 / 계정 / 기타
- 노출 순서 드래그 조정 / 비공개 여부 설정

**약관·개인정보처리방침**

- 서비스 이용약관 / 개인정보처리방침 / 전자금융거래이용약관
- 버전별 이력 관리 (이전 버전 열람 가능) / 시행일 설정
- 약관 개정 시 사용자 재동의 요청 여부 설정

### 11.13 어드민 ⑪ 사용자 관리 - 상세

**회원 조회**

- 검색: 이름·이메일·핸드폰번호·닉네임
- 필터: 가입일 / 가입 방법(이메일·카카오·네이버·구글) / 계정 상태(정상·정지·탈퇴)
- 테이블 컬럼: 이름 / 이메일 / 닉네임 / 가입일 / 가입 방법 / 계정 상태

**회원 상세**

- 기본 정보: 이름·이메일·닉네임·연락처·가입일·최근 로그인
- 예약 이력 요약: 총 예매 수·취소 수·총 결제 금액
- 보유 포인트·쿠폰 현황
- 본인인증 여부 (PASS 인증 완료 여부)

**계정 관리**

- 계정 정지: 사유 입력 → 정지 처리 → 사용자 이메일 안내 발송
- 정지 해제: 사유 기록 후 해제
- 포인트 수동 지급·차감: 사유 입력 필수 / 처리 로그 기록

### 11.14 어드민 ⑫ 시스템 설정 - 상세

**예약확정 자동 처리 설정**

- 이벤트 종료 후 자동 확정 기본 N일 설정 (기본 7일)
- 이벤트별 개별 N일 덮어쓰기 가능 (이벤트 개설 폼 ⑧에서 설정)
- 자동 확정 N일 전 사전 안내 알림 발송 여부 / 수단 설정

**대기열 설정**

- 기본 홀드 시간 설정 (분 단위, 이벤트별 덮어쓰기 가능)
- 대기열 최대 허용 인원 / 1계정 최대 기기 수 (기본 3기기)
- 대기열 활성화 기준 (동시 접속자 수 임계값 설정)

**알림 발송 규칙 관리**

- 알림 수단별 발송 활성화 여부 (이메일·SMS 글로벌 ON/OFF)
- 중복 발송 방지 기준 설정 (팔로우·오픈 알림 동시 발생 시 1회 발송)
- 발송 실패 시 재시도 횟수·간격 설정

**RBAC 역할 관리**

- 역할별 접근 권한 설정 / 담당자 계정에 역할 배정·해제
- 역할 변경 이력 로그

### 11.15 RBAC - 역할별 접근 범위 (전체 기준)

| 정산 담당자 | 정산 관리 전체 | V1 |
