# UI/UX 디자인 가이드

## 1. 디자인 원칙

참조 사이트(NOL 티켓) 분석에서 추출한 디자인 철학:

**1. 콘텐츠 우선 (Content-First)**
공연 포스터 이미지와 핵심 정보(제목, 일정, 가격)가 UI의 중심. 불필요한 장식적 요소를 최소화하고 콘텐츠가 돋보이도록 구성.

**2. 원스톱 효율성 (One-Stop Efficiency)**
예매 과정을 "좌석 선택 → 결제"로 최소화. `/onestop/*` URL 패턴이 이 철학을 반영. 불필요한 단계나 페이지 이동을 줄여 전환율 최적화.

**3. 신뢰와 안정감 (Trust & Stability)**
흰색 배경 + 짙은 텍스트의 깔끔한 조합. 결제/예매 관련 정보는 명확한 구조로 제시. 취소/환불 정책을 예매 과정에서 선제적으로 안내.

**4. 발견의 즐거움 (Serendipity)**
타임딜, 파이널콜, 얼리버드 등 다양한 프로모션 세그먼트로 "둘러보는 재미" 제공. 캐러셀과 탭을 활용한 콘텐츠 순환.

#### 다크모드

참조 사이트에서 다크모드 지원은 확인되지 않음. 향후 구현을 대비하여 다음을 권장:

- **CSS Custom Properties 기반 컬러 시스템:** 모든 컬러를 CSS 변수(`--color-primary`, `--color-bg` 등)로 정의하여 다크모드 전환 시 변수 값만 교체
- **Semantic 컬러 토큰 사용:** `#29292D` 같은 리터럴 값 대신 `--text-primary`, `--surface-bg` 같은 의미론적 토큰 사용
- **대비 비율 검증:** 다크 배경에서도 WCAG 2.1 AA 기준(4.5:1) 충족하는 팔레트 준비 필요

### 3.3 Card (공연 카드)

**구성요소:** 포스터 이미지, 배지(선택), 제목, 장소, 기간, 가격, 할인율

**변형:**
- **Standard Card:** 세로형, 포스터 + 텍스트 정보
- **TimeDeal Card:** 카운트다운 타이머 포함, 할인가/원가 표시
- **Ranking Card:** 순위 번호, 예매율 % 표시
- **FinalCall Card:** D-day 배지, 특가 할인율

```jsx
<PerformanceCard
  poster="/images/poster.jpg"
  title="뮤지컬 〈렘피카〉"
  venue="코엑스아티움 우리은행홀"
  period="2026.3.21~6.21"
  badges={[{ type: 'exclusive' }]}
  price={{ original: 170000, discounted: null }}
/>
```

### 3.4 Calendar (날짜 선택)

- 월간 달력 형태
- 네비게이션: `◀ 2026. 03 ▶`
- 예매 가능 날짜만 활성화 (클릭 가능)
- 선택된 날짜 강조 (Primary 색상)
- 오늘 날짜 표시
- 매진/판매종료 날짜는 비활성

### 3.5 SeatMap (좌석 배치도)

- SVG 기반 인터랙티브 좌석맵
- 등급별 색상 구분 (VIP, R, S, A 등)
- 좌석 상태: 선택가능(빈 좌석), 선택됨(내가 선택), 판매됨(비활성), 잠금(타인 선택중)
- 컨트롤: 확대(+), 축소(-), 전체보기(리셋)
- 핀치 줌 / 드래그 이동 지원
- 선택 시 사이드 패널에 좌석 정보 표시

> **관리자 좌석 배치도 편집기에 대한 참고:** `05-ADMIN-PREDICTION.md`에서 "SVG 기반 좌석맵 편집기"를 관리 기능으로 역추론했으나, 이는 별도 프로젝트급 공수(4~8주)가 예상됩니다. MVP에서는 외부 도구(Figma, Illustrator 등)에서 SVG를 제작하고 관리자 페이지에서 업로드하는 방식을 권장합니다. 인라인 편집기는 Phase 3 이후 검토합니다.

| MobileNav | GNB 탭 | 하단 탭 바 (홈, 검색, 마이페이지, 더보기) |
| SwipeableCard | PerformanceCard | 좌우 스와이프 제스처 지원 (캐러셀 대체) |
| PullToRefresh | 새로고침 버튼 | 당겨서 새로고침 (목록 페이지) |

> **참고:** 모바일 앱(Expo SDK 55)은 `02-PRD.md` Phase 4에서 개발 예정이며, 위 컴포넌트는 모바일 웹(반응형) 기준입니다. 네이티브 앱에서는 React Native 전용 컴포넌트로 별도 구현됩니다.

## 5. 인터랙션 패턴

### 5.1 좌석 선택 UX

```
1. 좌석 배치도 진입
   → 전체 보기로 초기 렌더링
   → 등급별 색상 범례 표시

2. 영역 탐색
   → 핀치/스크롤로 확대/축소
   → 등급 영역 클릭 시 해당 영역 줌인

3. 좌석 선택
   → 개별 좌석 탭/클릭
   → 선택 시: 색상 변경 + 사이드 패널에 추가
   → 이미 선택된 좌석 재클릭 시: 선택 해제
   → 타인이 선택한 좌석: 회색 비활성 + 선택 불가 피드백

4. 실시간 업데이트
   → WebSocket/SSE로 타 사용자 선택 상태 반영
   → 선택 → 점유 전환 시 애니메이션

5. 선택 완료
   → "선택 완료" 버튼 클릭
   → 취소/환불 안내 모달 (마감시간 안내)
   → "확인하고 예매하기" → 결제 페이지 이동
```

### 5.2 폼 유효성 검사 패턴

- **실시간 검증:** 입력 필드 blur 시 즉시 검증 (이름, 연락처, 이메일)
- **에러 표시:** 필드 하단 빨간색 에러 메시지 + 보더 색상 변경
- **필수 항목:** 라벨에 빨간 * 표시
- **제출 시:** 미입력 필드로 스크롤 + 포커스 이동

### 5.3 로딩 & 에러 상태

| 상태 | UI 패턴 |
|------|---------|
| 초기 로딩 | 스켈레톤 UI (카드, 리스트 형태) |
| 인라인 로딩 | 버튼 내 스피너 + 텍스트 변경 ("결제중...") |
| 페이지 전환 | 상단 프로그레스 바 (NProgress 스타일) |
| API 에러 | 인라인 에러 메시지 + 재시도 버튼 |
| 네트워크 에러 | 전체 화면 에러 페이지 + 재시도 |
| 매진 | 좌석 없음 안내 + 예매대기 등록 유도 |
| 대기열 | 순번 표시 + 프로그레스 바 + 예상 대기시간 |

### 5.4 토스트 & 알림

| Type | 위치 | 지속시간 | 용도 |
|------|------|----------|------|
| Success | 상단 중앙 | 3초 | 티켓캐스트 등록 완료 |
| Error | 상단 중앙 | 5초 | 좌석 선택 실패, 결제 실패 |
| Warning | 상단 중앙 | 5초 | 취소마감 임박 |
| Info | 상단 중앙 | 3초 | 일반 안내 |

## 6. 접근성 (a11y) 가이드라인

### 6.1 WCAG 2.1 AA 기준

| 항목 | 기준 | 구현 방법 |
|------|------|-----------|
| 색상 대비 | 최소 4.5:1 (일반 텍스트), 3:1 (대형 텍스트) | 디자인 토큰 검증 |
| 포커스 표시 | 모든 인터랙티브 요소에 가시적 포커스 링 | `outline: 2px solid #3549FF` |
| 대체 텍스트 | 모든 이미지에 alt 속성 | 포스터: 공연명, 아이콘: 기능 설명 |
| 랜드마크 | 시맨틱 HTML 사용 | `<header>`, `<nav>`, `<main>`, `<footer>` |
| 헤딩 구조 | 논리적 헤딩 순서 (h1 → h2 → h3) | 페이지당 h1 1개 |

### 6.2 키보드 네비게이션

- **Tab 순서:** 논리적 탭 순서 보장 (GNB → 콘텐츠 → Footer)
- **좌석 선택:** 방향키(↑↓←→)로 좌석 이동, Enter/Space로 선택
- **캘린더:** 방향키로 날짜 이동, Enter로 선택
- **모달:** Escape로 닫기, 포커스 트랩 적용
- **Skip Navigation:** "본문 바로가기" 링크 제공

### 6.3 스크린리더 대응

- **좌석 배치도:** `role="grid"`, 각 좌석에 `aria-label="A열 1번, VIP석, 170,000원, 선택 가능"`
- **캘린더:** `role="grid"`, 날짜에 `aria-label="3월 25일 화요일, 예매 가능"`
- **탭:** `role="tablist"`, `role="tab"`, `aria-selected`
- **배지:** `aria-label` 활용 (예: "단독판매 뮤지컬 렘피카")
- **카운트다운:** `aria-live="polite"`, 1분 간격 업데이트
- **대기열:** `aria-live="assertive"`, 순번 변경 시 알림
